# 常见问题

## AiShort 是 AI 提示词生成器吗？

不是。AiShort 是按场景**精选 + 整理**的人工 prompt 模板库，不是用 LLM 自动生成 prompt 的工具。每条 prompt 经过筛选并按标签分类，一键复制粘贴到 ChatGPT / Claude / Gemini / DeepSeek 等任意 AI 对话工具即可。如果你想要「从需求自动生成 prompt」那类工具，PromptPerfect、ChatGPT prompt generator 是另一类产品。

## AiShort 需要付费吗？需要 API Key 吗？

完全免费，**无需 API Key、无需注册**。浏览、搜索、复制提示词不要任何账号。

注册后可解锁收藏与拖拽排序、自定义标签、创建管理私人提示词、社区分享与投票、JSON 数据导出、跨设备同步等扩展功能——仍然免费。代码全部开源在 [GitHub](https://github.com/rockbenben/ChatGPT-Shortcut)。

注意：AiShort 只提供提示词，**调用 AI 模型本身是否收费由你使用的 AI 服务决定**（例如 ChatGPT 免费版/付费版、API 计费等）。

## 一共收录了多少提示词？覆盖哪些场景？

AiShort 累计收录 **5000+ 条提示词**，分两个库：

- **精选库** —— 人工筛选，按 25+ 场景标签分类，18 种语言翻译完整。覆盖标签：写作辅助、IT/编程、文章/报告、SEO/营销、企业职能、学术/教师、教育/学生、语言/翻译、心理/社交、思维训练、效率工具、终端/解释器、辩论/演讲、点评/评鉴、趣味科普、生活百科、医疗健康、金融顾问、音乐艺术、哲学/宗教、文本/词语、趣味游戏、专业顾问等。
- **[社区库](./community)** —— 占绝大多数，由用户持续投稿，可按热度或最新发布排序，支持搜索筛选；场景比精选库更细更广。

精选库搜不到合适的，去社区库再搜一次。

## 支持哪些 AI 模型？同一 prompt 跨模型通用吗？

AiShort 的提示词是通用的，**任何能输入提示词的 AI 场景都适用**——不只对话页，还包括 Cursor 等编程工具、API 调用、AI 代理等。下面是常见的对话类模型：

- 国际：ChatGPT、Gemini、Claude、Grok
- 国内：DeepSeek、通义千问、文心一言、豆包、Kimi、智谱清言、讯飞星火、腾讯元宝
- API 平台：OpenRouter、硅基流动、Groq

更多模型列表见 [开始上手 → 常用 AI 模型](./getting-started#常用-ai-模型)。

**跨模型通用性**：文本类 prompt 基本不挑模型——写作、翻译、编程、问答这类通用任务，同一份 prompt 在主流大语言模型上都能跑出有效结果，AiShort 每条 prompt 默认不绑定特定模型。唯一例外是绘图类 prompt（Midjourney / Stable Diffusion / DALL·E），需按各自语法调整。

## 为什么提示词用英文？

AI 模型对英文理解更准，输出更稳定。中文提示词也能用，但多次运行同一条中文提示词，结果差异会较大。重要场景建议用英文。

> 💡 想要中文回复？在提示词结尾加一句 `respond in Chinese`。

## 每次都要输入提示词吗？

**API 调用**：把提示词设为 `system prompt`，后续对话自动生效。

**网页版**：不切换对话的话，ChatGPT 会记住当前提示词。如果回复开始走偏，说明它「忘」了，重新粘贴一次即可。

**小技巧**：把常用对话保存为浏览器书签，下次直接打开。

## 怎么挑到合适的提示词？

按你要的输出反向找：要写文章用 [`write`](/?tags=write) 或 [`article`](/?tags=article) 标签；要写代码用 [`code`](/?tags=code) 或 [`interpreter`](/?tags=interpreter)；要翻译用 [`language`](/?tags=language)；做角色扮演用 [`games`](/?tags=games)；不熟悉标签时直接关键词搜索。每条 prompt 卡片显示热度（复制次数），复制多的通常质量更稳。

## 为什么搜索不到相关的提示词？

首页搜索范围是**精选提示词库**（登录后还包含你的个人提示词），**不包含**社区分享的提示词。

如果用短关键词在首页搜不到，去 [社区提示词](./community) 页面再搜一次——那里有更多用户分享内容。

## AI 输出了虚假信息怎么办？

AI 有时会产生「幻觉」，输出看似合理但实际错误的信息。处理方法：

1. **核实关键信息**：特别是数据、引用、代码等
2. **多轮优化**：让 AI 对回答进行再次检查和优化
3. **交叉验证**：用不同的提示词或模型验证重要结论

合适的提示词能减少 AI 幻觉。

## 提示词效果不理想怎么调？

不要直接换提示词，先从这几个方向调：

1. **把方括号 `[占位符]` 改得更具体**——加上风格、长度、领域、读者画像等细节
2. **追问 AI 优化**：对不满意的回复追问「请改得更 X」或「请用 Y 风格重写」，AI 通常一两轮迭代后就能贴近目标，也可让 AI 对自己的回答打分并改进
3. **切换模型对比**：同一 prompt 换个模型（Claude / ChatGPT / Gemini / DeepSeek 等）再试试，不同模型各有所长，效果可能差别不小
4. **试试社区**：[社区提示词](./community) 里可能有更适合的版本——也欢迎把你的好提示词分享到那里
5. **反馈建议**：遇到问题或有改进建议？欢迎 [反馈](/feedback)

## 如何备份我的提示词？

1. 进入「我的账户」→ 找到「数据管理 → 导出提示词」一栏
2. 点「导出数据」按钮
3. 系统自动生成 JSON 文件下载

建议定期备份，防止数据丢失。

## AiShort 上的提示词可以用于商业项目吗？

可以，但分情况看授权：

1. 来自 [Awesome ChatGPT Prompts](https://github.com/f/awesome-chatgpt-prompts) 的内容遵循 **CC0 公共领域**，无任何限制
2. AiShort 自建与社区投稿内容默认采用 **CC BY-SA 4.0** 开放许可——商用、二次分发、改编都允许，要求保留 AiShort / 原作者署名并允许下游同样开放
3. 你**用 prompt 生成的 AI 内容**版权按你所用 AI 服务商（OpenAI / Anthropic / Google 等）的使用条款归属，多数情况归你
4. prompt 卡片说明里明确标注「仅个人用」的极少数条目除外

## 用 AiShort 复制提示词会泄露我的数据吗？

不会。AiShort 的提示词以静态 JSON 形式打包进站点资源，**复制动作本身在你浏览器本地剪贴板完成**。

你**填入方括号占位符的具体内容、AI 给你的回复**——这些都直接在你与所选 AI 平台（ChatGPT / Claude / Gemini / DeepSeek 等）之间传输，AiShort 看不到。

需要说明的是：复制时会向后端发送**匿名计数事件**（POST `/cards/<id>/copy`），仅用于统计每条提示词的热度（卡片上显示的「复制次数」）。**只传卡片 ID，不包含**你填的内容、不包含个人信息、不关联用户身份。

**登录账号后**的收藏列表、自定义提示词、社区投稿才会同步到后端用于跨设备同步，且支持随时 JSON 导出与一键删除全部数据。
