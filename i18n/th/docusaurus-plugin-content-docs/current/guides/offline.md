# เวอร์ชันติดตั้งแบบออฟไลน์

**สถานการณ์ที่เหมาะสม**: อินทราเน็ตองค์กร เครือข่ายราชการ สภาพแวดล้อมที่เป็นความลับ เครือข่ายมหาวิทยาลัย และสถานการณ์อื่นๆ ที่**ไม่สามารถหรือไม่สะดวกเข้าถึงเครือข่ายภายนอก**

AI Short เวอร์ชันออฟไลน์ไม่ต้องใช้เซิร์ฟเวอร์ backend และไม่ต้องลงทะเบียนผู้ใช้ ข้อมูลทั้งหมดเก็บในเบราว์เซอร์ หลังจากติดตั้งแล้ว ทีมสามารถใช้งานได้โดยตรงในอินทราเน็ต

## วิธีการใช้งานสำหรับทีม

เวอร์ชันออฟไลน์เป็นเว็บไซต์แบบ static ล้วน หลังจากติดตั้งบนเซิร์ฟเวอร์อินทราเน็ตแล้ว สมาชิกในทีมเพียงเปิดเบราว์เซอร์ไปที่ที่อยู่อินทราเน็ตนั้นก็สามารถใช้งานได้:

1. **ผู้ดูแลระบบ** ติดตั้งเวอร์ชันออฟไลน์บนเซิร์ฟเวอร์อินทราเน็ต (เช่น `http://192.168.1.100:3000`)
2. **สมาชิกในทีม** เปิดเบราว์เซอร์ไปที่ที่อยู่นั้น สามารถเรียกดู ค้นหา และคัดลอก prompt ได้
3. **คอลเลกชันและ prompt ที่สร้างเอง** ของแต่ละคนจะถูกบันทึกไว้ในเบราว์เซอร์ของตัวเอง ไม่ส่งผลกระทบต่อกัน
4. ไม่ต้องลงทะเบียนบัญชี ไม่ต้องติดตั้งซอฟต์แวร์ใดๆ เปิดแล้วใช้งานได้เลย

```
เซิร์ฟเวอร์อินทราเน็ต (ติดตั้งเวอร์ชันออฟไลน์)
   ├── ข้อมูลคลัง prompt (แชร์สำหรับทุกคน จาก static JSON)
   │
   ├── เบราว์เซอร์ผู้ใช้ A → localStorage (คอลเลกชันและ prompt ที่สร้างเองของ A)
   ├── เบราว์เซอร์ผู้ใช้ B → localStorage (คอลเลกชันและ prompt ที่สร้างเองของ B)
   └── เบราว์เซอร์ผู้ใช้ C → localStorage (คอลเลกชันและ prompt ที่สร้างเองของ C)
```

:::tip เคล็ดลับ
เนื้อหาคลัง prompt (prompt ที่คัดสรร) เป็นข้อมูล static ที่แพ็คในเวลาบิลด์ ผู้ใช้ทุกคนเห็นเนื้อหาเดียวกัน คอลเลกชัน prompt ที่สร้างเอง การจัดเรียง และแท็กของแต่ละผู้ใช้จะถูกบันทึกใน localStorage ของเบราว์เซอร์ของตัวเอง เป็นอิสระจากกัน
:::

## ความแตกต่างกับเวอร์ชันออนไลน์

| ฟีเจอร์ | เวอร์ชันออนไลน์ | เวอร์ชันออฟไลน์ |
| ---- | ------ | ------ |
| เรียกดู/ค้นหา/กรอง prompt | ✅ | ✅ |
| คัดลอก prompt | ✅ | ✅ |
| จัดการคอลเลกชัน | เก็บบนเซิร์ฟเวอร์ | เก็บในเบราว์เซอร์ |
| prompt ที่กำหนดเอง | เก็บบนเซิร์ฟเวอร์ | เก็บในเบราว์เซอร์ |
| คอลเลกชันของฉัน (ลากจัดเรียง, แท็ก) | ✅ | ✅ |
| รองรับหลายภาษา (18 ภาษา) | ✅ | ✅ |
| นำเข้า/ส่งออกข้อมูล | ✅ | ✅ (รูปแบบเข้ากันได้) |
| หน้ารายละเอียด prompt | ✅ | ✅ (ข้อมูล static ไม่มีความคิดเห็น) |
| ลงทะเบียน/เข้าสู่ระบบ | ✅ | ❌ (ไม่ต้องใช้บัญชี) |
| รายการ prompt ชุมชน/โหวต | ✅ | ❌ |
| ความคิดเห็นและข้อเสนอแนะ | ✅ | ❌ |

## การจัดเก็บข้อมูล

ข้อมูลของผู้ใช้แต่ละคนถูกบันทึกใน localStorage ของ**เบราว์เซอร์ของตัวเอง** ไม่เกี่ยวข้องกับเซิร์ฟเวอร์:

| ข้อมูล | คีย์จัดเก็บ | คำอธิบาย |
| ---- | ------ | ---- |
| รายการคอลเลกชัน | `local_favorites` | อาร์เรย์ ID ของ prompt ที่เก็บ |
| prompt ที่สร้างเอง | `local_user_prompts` | ข้อมูล prompt ที่ผู้ใช้สร้าง |
| ลำดับการจัดเรียง | `local_myspace_order` | ลำดับการ์ดในคอลเลกชันของฉัน |
| แท็กที่กำหนดเอง | `local_custom_tags` | คำจำกัดความแท็กและความสัมพันธ์การกำหนด |

:::caution ข้อควรระวัง
- localStorage ของเบราว์เซอร์มีขีดจำกัดความจุประมาณ 5MB เพียงพอสำหรับการใช้งานประจำวัน
- การล้างข้อมูลเบราว์เซอร์จะทำให้ข้อมูลส่วนตัวสูญหาย แนะนำให้สำรองข้อมูลเป็นประจำผ่าน "ตั้งค่า > ส่งออกข้อมูล"
- หลังจากเปลี่ยนคอมพิวเตอร์หรือเบราว์เซอร์ ต้องนำเข้าข้อมูลใหม่
:::

## การติดตั้ง

เวอร์ชันออฟไลน์อิงจากสาขา `offline` หลังจากผู้ดูแลระบบติดตั้งเสร็จครั้งเดียว สมาชิกในทีมไม่ต้องดำเนินการใดๆ

### ติดตั้งด้วย Docker (แนะนำ)

วิธีติดตั้งที่ง่ายที่สุด คำสั่งเดียวก็สามารถรันบนเซิร์ฟเวอร์อินทราเน็ตได้:

```bash
# ใช้อิมเมจเวอร์ชันออฟไลน์ที่สร้างไว้ล่วงหน้า
docker run -d -p 3000:3000 --name aishort-offline ghcr.io/rockbenben/chatgpt-shortcut:offline

# หรือใช้ Docker Hub
docker run -d -p 3000:3000 --name aishort-offline rockben/chatgpt-shortcut:offline
```

หลังจากติดตั้งแล้ว สมาชิกในทีมเข้าถึง `http://<IP เซิร์ฟเวอร์>:3000` เพื่อใช้งาน

ใช้ `docker-compose`:

```yml
services:
  aishort-offline:
    container_name: aishort-offline
    image: ghcr.io/rockbenben/chatgpt-shortcut:offline
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### บิลด์จากซอร์สโค้ด

หากต้องการปรับแต่งเนื้อหา prompt หรือแก้ไขการกำหนดค่า:

```bash
# โคลนสาขาออฟไลน์
git clone -b offline https://github.com/rockbenben/ChatGPT-Shortcut.git
cd ChatGPT-Shortcut

# ติดตั้ง dependency
yarn

# พัฒนาในเครื่อง
yarn start

# บิลด์เวอร์ชันภาษาเดียว (จีน)
yarn build --locale zh-Hans

# บิลด์ทุกภาษา
yarn build
```

ผลลัพธ์การบิลด์อยู่ในไดเรกทอรี `build/` สามารถติดตั้งบนเซิร์ฟเวอร์ไฟล์ static ใดก็ได้ (Nginx, Apache, Caddy ฯลฯ)

### ตัวอย่างการกำหนดค่า Nginx

```nginx
server {
    listen 3000;
    server_name _;
    root /path/to/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### ติดตั้งบนแพลตฟอร์ม

เมื่อติดตั้งบน Vercel, Cloudflare Pages และแพลตฟอร์มอื่นๆ ให้เลือกสาขา `offline` ขั้นตอนอื่นๆ เหมือนกับเวอร์ชันออนไลน์ ดูรายละเอียดที่ [การติดตั้งโปรเจกต์](../deploy)

## นำเข้า-ส่งออกข้อมูล

### ส่งออก

ไปที่ "ตั้งค่า > ส่งออกข้อมูล" ส่งออกคอลเลกชันส่วนตัวและ prompt ที่สร้างเองเป็นไฟล์ JSON

### นำเข้า

รองรับการนำเข้าไฟล์ JSON ในรูปแบบต่อไปนี้:

- **ไฟล์ที่ส่งออกจากเวอร์ชันออฟไลน์**: กู้คืนคอลเลกชัน prompt การจัดเรียง และแท็กอย่างสมบูรณ์
- **ไฟล์ที่ส่งออกจากเวอร์ชันออนไลน์**: เข้ากันได้โดยอัตโนมัติ
  - prompt ผู้ใช้ → รวมเข้ากับข้อมูลท้องถิ่น (ลบซ้ำตามชื่อ)
  - คอลเลกชันที่คัดสรร (card) → รวมเข้ากับคอลเลกชันท้องถิ่น
  - คอลเลกชันชุมชน (community) → แปลงเป็น prompt ที่สร้างเองในเครื่องโดยอัตโนมัติ
  - การจัดเรียง MySpace → กู้คืนไปยังท้องถิ่น
  - แท็กที่กำหนดเอง → เพิ่มและรวม (ไม่เขียนทับที่มีอยู่)

### การย้ายจากเวอร์ชันออนไลน์

1. ส่งออกข้อมูลจากหน้า "บัญชีของฉัน" ในเวอร์ชันออนไลน์ (aishort.top)
2. นำเข้าไฟล์ JSON นั้นในหน้า "ตั้งค่า" ของเวอร์ชันออฟไลน์
3. คอลเลกชันชุมชนจะถูกแปลงเป็น prompt ท้องถิ่นโดยอัตโนมัติ คอลเลกชันที่คัดสรรจะซิงค์ตามปกติ

## คำถามที่พบบ่อย

### หลังจากติดตั้งแล้ว ทีมใช้งานอย่างไร?

หลังจากผู้ดูแลระบบติดตั้งบนเซิร์ฟเวอร์อินทราเน็ตแล้ว แจ้งที่อยู่เข้าถึง (เช่น `http://192.168.1.100:3000`) ให้สมาชิกในทีม ทุกคนเปิดเบราว์เซอร์ ไม่ต้องติดตั้ง ไม่ต้องลงทะเบียน

### ข้อมูลของแต่ละคนจะส่งผลกระทบต่อกันหรือไม่?

ไม่ คอลเลกชันและ prompt ที่สร้างเองของแต่ละคนถูกบันทึกใน localStorage ของเบราว์เซอร์ของตัวเอง เป็นอิสระอย่างสมบูรณ์ บนเซิร์ฟเวอร์มีเพียงคลัง prompt ที่แชร์ (อ่านอย่างเดียว) เท่านั้น

### ข้อมูลจะสูญหายได้หรือไม่?

การดำเนินการต่อไปนี้อาจทำให้ข้อมูลส่วนตัวสูญหาย:

- ล้างข้อมูล/แคชของเบราว์เซอร์
- ใช้โหมดส่วนตัว/ไม่ระบุตัวตน
- เปลี่ยนคอมพิวเตอร์หรือเบราว์เซอร์

แนะนำให้สำรองข้อมูลสำคัญเป็นประจำผ่าน "ตั้งค่า > ส่งออกข้อมูล" เป็นไฟล์ JSON

### สามารถแชร์ prompt ที่สร้างเองระหว่างทีมได้หรือไม่?

ได้ คนหนึ่งส่งออกไฟล์ JSON แล้วสมาชิกคนอื่นนำเข้าใน "ตั้งค่า > นำเข้าข้อมูล" ระบบจะลบซ้ำโดยอัตโนมัติเมื่อนำเข้า

### จะอัปเดตคลัง prompt ได้อย่างไร?

คลัง prompt เป็นข้อมูล static ที่แพ็คในเวลาบิลด์ วิธีอัปเดต:

1. ผู้ดูแลระบบดึงโค้ดสาขา `offline` ล่าสุด
2. บิลด์ใหม่และติดตั้ง (หรือดึง Docker image ล่าสุด)
3. สมาชิกในทีมรีเฟรชหน้าเบราว์เซอร์เพื่อดูเนื้อหาใหม่ (ข้อมูลส่วนตัวไม่ได้รับผลกระทบ)

### รูปแบบข้อมูลของเวอร์ชันออฟไลน์เข้ากันได้กับเวอร์ชันออนไลน์หรือไม่?

เข้ากันได้ รูปแบบ JSON ที่ส่งออกเหมือนกัน สามารถนำเข้า-ส่งออกระหว่างสองเวอร์ชันได้ ID ของ prompt ในสองเวอร์ชันแตกต่างกัน (เวอร์ชันออนไลน์ใช้ ID เซิร์ฟเวอร์ เวอร์ชันออฟไลน์ใช้ ID timestamp) แต่เมื่อนำเข้าจะลบซ้ำตามชื่อ ไม่มีความขัดแย้ง
