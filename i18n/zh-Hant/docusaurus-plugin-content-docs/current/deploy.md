# 項目部署

> **目標讀者**：想自部署或定制 AiShort 的開發者。普通用戶直接用 [aishort.top](https://www.aishort.top) 即可，不需要看本文。

## 選擇部署形態

先按需求選一種部署形態：

| 形態                   | 後端                     | 說明                                                                                     |
| ---------------------- | ------------------------ | ---------------------------------------------------------------------------------------- |
| **標準部署**（預設）   | 復用官方共享後端         | fork 後可自定義網站名稱、描述、提示詞等（見[配置與自定義](./deploy/configuration)）；登入、收藏、社群、同步開箱即用 |
| **離線部署版**         | 無後端，資料存瀏覽器本地 | 企業內網、政務等無外網環境，無需帳號                                                      |
| **帶後端的完整自部署** | 自建獨立後端             | 需要獨立的帳號體系、資料自主可控、私有社群                                                |

前兩種照本文操作即可。第三種因後端服務並未開源，請[郵件聯繫開發者](mailto:qingwhat@gmail.com)，簡單說明你的使用場景與規模，即可獲取部署方案與支援。

## 部署文件

按部署流程拆分為以下幾篇，按需查閱：

- **[標準部署](./deploy/standard)** —— 復用官方共享後端，支持本地建置、Vercel、Cloudflare Pages、Docker 四種方式。
- **[離線部署版](./deploy/offline)** —— 企業內網、政務網路等無外網環境的離線方案，無需後端與帳號。
- **[配置與自定義](./deploy/configuration)** —— 修改網站標題、描述、提示詞，以及對接自定義後端。
- **[開啟同步更新](./deploy/sync-updates)** —— 讓 fork 跟隨上游自動更新，避免功能落後。
