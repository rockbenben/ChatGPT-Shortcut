# نسخة النشر غير المتصلة

**السيناريوهات المناسبة**: الشبكة الداخلية للمؤسسات، الشبكات الحكومية، البيئات السرية، شبكات الحرم الجامعي وغيرها من السيناريوهات التي **لا يمكن فيها أو يصعب الوصول إلى الشبكة الخارجية**.

لا تحتاج النسخة غير المتصلة من AI Short إلى خادم خلفي أو تسجيل مستخدم، جميع البيانات مخزنة محليًا في المتصفح. بعد النشر، يمكن للفريق الاستخدام مباشرة في الشبكة الداخلية.

## طريقة استخدام الفريق

النسخة غير المتصلة هي موقع ثابت بالكامل، بعد نشره على خادم الشبكة الداخلية، يحتاج أعضاء الفريق فقط لفتح المتصفح والذهاب إلى عنوان الشبكة الداخلية لاستخدامه:

1. **المسؤول** ينشر النسخة غير المتصلة على خادم الشبكة الداخلية (مثل `http://192.168.1.100:3000`)
2. **أعضاء الفريق** يفتحون المتصفح إلى ذلك العنوان، يمكنهم تصفح المطالبات والبحث عنها ونسخها
3. **المفضلة والمطالبات المخصصة** لكل شخص محفوظة في متصفحه الخاص، لا يؤثر بعضها على بعض
4. لا حاجة لتسجيل حساب، لا حاجة لتثبيت أي برنامج، افتح واستخدم

```
خادم الشبكة الداخلية (نشر النسخة غير المتصلة)
   ├── بيانات مكتبة المطالبات (مشتركة للجميع، من JSON ثابت)
   │
   ├── متصفح المستخدم A ← localStorage (مفضلات ومطالبات مخصصة لـ A)
   ├── متصفح المستخدم B ← localStorage (مفضلات ومطالبات مخصصة لـ B)
   └── متصفح المستخدم C ← localStorage (مفضلات ومطالبات مخصصة لـ C)
```

:::tip نصيحة
محتوى مكتبة المطالبات (المطالبات المختارة) هو بيانات ثابتة محزومة وقت البناء، يرى جميع المستخدمين نفس المحتوى. مفضلات كل مستخدم، المطالبات المخصصة، الترتيب والعلامات محفوظة في localStorage لمتصفحه الخاص، مستقلة عن بعضها.
:::

## الفرق مع النسخة المتصلة

| الميزة | النسخة المتصلة | النسخة غير المتصلة |
| ---- | ------ | ------ |
| تصفح/بحث/تصفية المطالبات | ✅ | ✅ |
| نسخ المطالبات | ✅ | ✅ |
| إدارة المفضلة | تخزين على الخادم | تخزين محلي في المتصفح |
| مطالبات مخصصة | تخزين على الخادم | تخزين محلي في المتصفح |
| مجموعتي (سحب وإفلات، علامات) | ✅ | ✅ |
| دعم متعدد اللغات (18 لغة) | ✅ | ✅ |
| استيراد/تصدير البيانات | ✅ | ✅ (التنسيق متوافق) |
| صفحة تفاصيل المطالبة | ✅ | ✅ (بيانات ثابتة، بدون تعليقات) |
| تسجيل/تسجيل دخول المستخدم | ✅ | ❌ (لا حاجة لحساب) |
| قائمة مطالبات المجتمع/التصويت | ✅ | ❌ |
| التعليقات والملاحظات | ✅ | ❌ |

## تخزين البيانات

بيانات كل مستخدم محفوظة في localStorage في **متصفحه الخاص**، لا علاقة لها بالخادم:

| البيانات | مفتاح التخزين | الوصف |
| ---- | ------ | ---- |
| قائمة المفضلة | `local_favorites` | مصفوفة معرفات المطالبات المفضلة |
| المطالبات المخصصة | `local_user_prompts` | بيانات المطالبات التي أنشأها المستخدم |
| ترتيب الفرز | `local_myspace_order` | ترتيب البطاقات في مجموعتي |
| العلامات المخصصة | `local_custom_tags` | تعريف العلامات وعلاقات التعيين |

:::caution تنبيه
- تخزين المتصفح المحلي له حد سعة حوالي 5 ميجابايت، كافٍ تمامًا للاستخدام اليومي.
- مسح بيانات المتصفح سيؤدي إلى فقدان البيانات الشخصية، يُنصح بالنسخ الاحتياطي المنتظم عبر "الإعدادات > تصدير البيانات".
- بعد تغيير الكمبيوتر أو المتصفح، يجب إعادة استيراد البيانات.
:::

## النشر

النسخة غير المتصلة مبنية على فرع `offline`. بعد إكمال المسؤول للنشر مرة واحدة، لا يحتاج أعضاء الفريق للقيام بأي إجراء.

### نشر Docker (موصى به)

أبسط طريقة نشر، أمر واحد يمكن تشغيله على خادم الشبكة الداخلية:

```bash
# استخدام صورة النسخة غير المتصلة المبنية مسبقًا
docker run -d -p 3000:3000 --name aishort-offline ghcr.io/rockbenben/chatgpt-shortcut:offline

# أو استخدام Docker Hub
docker run -d -p 3000:3000 --name aishort-offline rockben/chatgpt-shortcut:offline
```

بعد النشر، يصل أعضاء الفريق إلى `http://<عنوان IP الخادم>:3000` للاستخدام.

استخدام `docker-compose`:

```yml
services:
  aishort-offline:
    container_name: aishort-offline
    image: ghcr.io/rockbenben/chatgpt-shortcut:offline
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### البناء من الكود المصدري

إذا كنت بحاجة لتخصيص محتوى المطالبات أو تعديل التكوين:

```bash
# استنساخ فرع offline
git clone -b offline https://github.com/rockbenben/ChatGPT-Shortcut.git
cd ChatGPT-Shortcut

# تثبيت التبعيات
yarn

# التطوير المحلي
yarn start

# بناء نسخة لغة واحدة (الصينية)
yarn build --locale zh-Hans

# بناء جميع اللغات
yarn build
```

مخرجات البناء في دليل `build/`، يمكن نشرها على أي خادم ملفات ثابت (Nginx، Apache، Caddy، إلخ).

### مثال تكوين Nginx

```nginx
server {
    listen 3000;
    server_name _;
    root /path/to/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### النشر على المنصات

عند النشر على Vercel و Cloudflare Pages ومنصات أخرى، اختر فرع `offline`، الخطوات الأخرى مطابقة للنسخة المتصلة، انظر التفاصيل في [نشر المشروع](../deploy).

## استيراد وتصدير البيانات

### التصدير

اذهب إلى "الإعدادات > تصدير البيانات"، صدّر المفضلة الشخصية والمطالبات المخصصة كملف JSON.

### الاستيراد

يدعم استيراد ملفات JSON بالتنسيقات التالية:

- **ملف مصدّر من النسخة غير المتصلة**: استعادة كاملة للمفضلة والمطالبات والترتيب والعلامات
- **ملف مصدّر من النسخة المتصلة**: متوافق تلقائيًا
  - مطالبات المستخدم → دمج محليًا (إزالة التكرار حسب العنوان)
  - المفضلة المختارة (card) → دمج في المفضلة المحلية
  - مفضلة المجتمع (community) → تحويل تلقائي إلى مطالبات مخصصة محلية
  - ترتيب MySpace → استعادة محليًا
  - العلامات المخصصة → إضافة ودمج (لا تستبدل الموجودة)

### الانتقال من النسخة المتصلة

1. صدّر البيانات من صفحة "حسابي" في النسخة المتصلة (aishort.top)
2. استورد ملف JSON هذا في صفحة "الإعدادات" للنسخة غير المتصلة
3. ستتحول مفضلة المجتمع تلقائيًا إلى مطالبات محلية، والمفضلة المختارة تتزامن بشكل طبيعي

## الأسئلة الشائعة

### كيف يستخدم الفريق بعد النشر؟

بعد نشر المسؤول على خادم الشبكة الداخلية، أخبر أعضاء الفريق بعنوان الوصول (مثل `http://192.168.1.100:3000`). كل شخص يفتح المتصفح، لا حاجة للتثبيت، لا حاجة للتسجيل.

### هل ستؤثر بيانات كل شخص على الآخرين؟

لا. مفضلات كل شخص ومطالباته المخصصة محفوظة في localStorage لمتصفحه الخاص، مستقلة تمامًا. على الخادم توجد فقط مكتبة المطالبات المشتركة (للقراءة فقط).

### هل يمكن أن تُفقد البيانات؟

العمليات التالية قد تؤدي إلى فقدان البيانات الشخصية:

- مسح بيانات/ذاكرة التخزين المؤقت للمتصفح
- استخدام وضع التصفح الخاص/المتخفي
- تغيير الكمبيوتر أو المتصفح

يُنصح بعمل نسخة احتياطية منتظمة للبيانات المهمة عبر "الإعدادات > تصدير البيانات" كملف JSON.

### هل يمكن مشاركة المطالبات المخصصة بين الفريق؟

نعم. يقوم شخص واحد بتصدير ملف JSON، ثم يستورده الأعضاء الآخرون في "الإعدادات > استيراد البيانات". يتم إزالة التكرار تلقائيًا عند الاستيراد.

### كيف يتم تحديث مكتبة المطالبات؟

مكتبة المطالبات هي بيانات ثابتة محزومة وقت البناء. طريقة التحديث:

1. يسحب المسؤول أحدث كود فرع `offline`
2. يعيد البناء والنشر (أو سحب أحدث صورة Docker)
3. يحدث أعضاء الفريق صفحة المتصفح لرؤية المحتوى الجديد (البيانات الشخصية لا تتأثر)

### هل تنسيق بيانات النسخة غير المتصلة متوافق مع النسخة المتصلة؟

متوافق. تنسيق JSON المصدّر متطابق، يمكن الاستيراد والتصدير بين النسختين. معرفات المطالبات مختلفة في النسختين (المتصلة تستخدم معرف الخادم، غير المتصلة تستخدم معرف الطابع الزمني)، لكن عند الاستيراد يتم إزالة التكرار حسب العنوان، لا يوجد تعارض.
