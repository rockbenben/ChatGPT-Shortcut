# অফলাইন ডিপ্লয়মেন্ট সংস্করণ

**প্রযোজ্য পরিস্থিতি**: এন্টারপ্রাইজ ইন্ট্রানেট, সরকারি নেটওয়ার্ক, গোপনীয় পরিবেশ, ক্যাম্পাস নেটওয়ার্ক ইত্যাদি **যেখানে বাহ্যিক নেটওয়ার্কে অ্যাক্সেস সম্ভব নয় বা অসুবিধাজনক**।

AI Short অফলাইন সংস্করণে কোনো ব্যাকএন্ড সার্ভার বা ব্যবহারকারী নিবন্ধনের প্রয়োজন নেই, সমস্ত ডেটা ব্রাউজারে স্থানীয়ভাবে সংরক্ষিত হয়। ডিপ্লয়মেন্টের পরে দল ইন্ট্রানেটে সরাসরি ব্যবহার করতে পারে।

## দল ব্যবহারের পদ্ধতি

অফলাইন সংস্করণ একটি সম্পূর্ণ স্ট্যাটিক ওয়েবসাইট, ইন্ট্রানেট সার্ভারে ডিপ্লয় করার পরে, দলের সদস্যরা কেবল ব্রাউজার দিয়ে সেই ইন্ট্রানেট ঠিকানায় গিয়ে ব্যবহার করতে পারবেন:

1. **অ্যাডমিন** অফলাইন সংস্করণ ইন্ট্রানেট সার্ভারে ডিপ্লয় করেন (যেমন `http://192.168.1.100:3000`)
2. **দলের সদস্যরা** ব্রাউজার দিয়ে সেই ঠিকানা খুলুন, প্রম্পট ব্রাউজ, অনুসন্ধান এবং কপি করতে পারেন
3. প্রতিটি ব্যক্তির **পছন্দ এবং কাস্টম প্রম্পট তাদের নিজের ব্রাউজারে সংরক্ষিত** হয়, একে অপরকে প্রভাবিত করে না
4. অ্যাকাউন্ট নিবন্ধনের প্রয়োজন নেই, কোনো সফটওয়্যার ইনস্টল করার প্রয়োজন নেই, খুলুন এবং ব্যবহার করুন

```
ইন্ট্রানেট সার্ভার (অফলাইন সংস্করণ ডিপ্লয়)
   ├── প্রম্পট লাইব্রেরি ডেটা (সকলের জন্য শেয়ার, স্ট্যাটিক JSON থেকে)
   │
   ├── ব্যবহারকারী A এর ব্রাউজার → localStorage (A এর পছন্দ এবং কাস্টম প্রম্পট)
   ├── ব্যবহারকারী B এর ব্রাউজার → localStorage (B এর পছন্দ এবং কাস্টম প্রম্পট)
   └── ব্যবহারকারী C এর ব্রাউজার → localStorage (C এর পছন্দ এবং কাস্টম প্রম্পট)
```

:::tip পরামর্শ
প্রম্পট লাইব্রেরির বিষয়বস্তু (নির্বাচিত প্রম্পট) হলো বিল্ড সময়ে প্যাক করা স্ট্যাটিক ডেটা, সমস্ত ব্যবহারকারী একই বিষয়বস্তু দেখেন। প্রতিটি ব্যবহারকারীর পছন্দ, কাস্টম প্রম্পট, সাজানো এবং ট্যাগ তাদের নিজস্ব ব্রাউজারের localStorage-এ সংরক্ষিত, পরস্পর স্বতন্ত্র।
:::

## অনলাইন সংস্করণের সাথে পার্থক্য

| বৈশিষ্ট্য | অনলাইন সংস্করণ | অফলাইন সংস্করণ |
| ---- | ------ | ------ |
| প্রম্পট ব্রাউজ/অনুসন্ধান/ফিল্টার | ✅ | ✅ |
| প্রম্পট কপি | ✅ | ✅ |
| পছন্দ ব্যবস্থাপনা | সার্ভার স্টোরেজ | ব্রাউজার লোকাল স্টোরেজ |
| কাস্টম প্রম্পট | সার্ভার স্টোরেজ | ব্রাউজার লোকাল স্টোরেজ |
| আমার সংগ্রহ (ড্র্যাগ সর্টিং, ট্যাগ) | ✅ | ✅ |
| মাল্টি-ল্যাঙ্গুয়েজ সাপোর্ট (১৮ ভাষা) | ✅ | ✅ |
| ডেটা ইম্পোর্ট/এক্সপোর্ট | ✅ | ✅ (ফরম্যাট সামঞ্জস্যপূর্ণ) |
| প্রম্পট বিস্তারিত পৃষ্ঠা | ✅ | ✅ (স্ট্যাটিক ডেটা, মন্তব্য নেই) |
| ব্যবহারকারী নিবন্ধন/লগইন | ✅ | ❌ (অ্যাকাউন্টের প্রয়োজন নেই) |
| কমিউনিটি প্রম্পট তালিকা/ভোটিং | ✅ | ❌ |
| মন্তব্য প্রতিক্রিয়া | ✅ | ❌ |

## ডেটা স্টোরেজ

প্রতিটি ব্যবহারকারীর ডেটা **তাদের নিজস্ব ব্রাউজারের** localStorage-এ সংরক্ষিত, সার্ভারের সাথে কোনো সম্পর্ক নেই:

| ডেটা | স্টোরেজ কী | বিবরণ |
| ---- | ------ | ---- |
| পছন্দ তালিকা | `local_favorites` | পছন্দের প্রম্পট ID অ্যারে |
| কাস্টম প্রম্পট | `local_user_prompts` | ব্যবহারকারী তৈরি প্রম্পট ডেটা |
| সাজানোর ক্রম | `local_myspace_order` | আমার সংগ্রহে কার্ডের ক্রম |
| কাস্টম ট্যাগ | `local_custom_tags` | ট্যাগ সংজ্ঞা এবং বরাদ্দ সম্পর্ক |

:::caution সতর্কতা
- ব্রাউজার লোকাল স্টোরেজের প্রায় 5MB ক্ষমতা সীমা আছে, দৈনিক ব্যবহারের জন্য সম্পূর্ণ পর্যাপ্ত।
- ব্রাউজার ডেটা মুছে ফেললে ব্যক্তিগত ডেটা হারিয়ে যাবে, নিয়মিত "সেটিংস > ডেটা এক্সপোর্ট" এর মাধ্যমে ব্যাকআপ নেওয়ার পরামর্শ দেওয়া হয়।
- কম্পিউটার বা ব্রাউজার পরিবর্তন করার পরে ডেটা পুনরায় ইম্পোর্ট করতে হবে।
:::

## ডিপ্লয়মেন্ট

অফলাইন সংস্করণ `offline` ব্রাঞ্চের উপর ভিত্তি করে। অ্যাডমিন একবার ডিপ্লয়মেন্ট সম্পন্ন করলে, দলের সদস্যদের কোনো পদক্ষেপ নিতে হবে না।

### Docker ডিপ্লয়মেন্ট (প্রস্তাবিত)

সবচেয়ে সহজ ডিপ্লয়মেন্ট পদ্ধতি, এক লাইন কমান্ডে ইন্ট্রানেট সার্ভারে চালানো যায়:

```bash
# পূর্ব-নির্মিত অফলাইন সংস্করণ ইমেজ ব্যবহার করুন
docker run -d -p 3000:3000 --name aishort-offline ghcr.io/rockbenben/chatgpt-shortcut:offline

# অথবা Docker Hub ব্যবহার করুন
docker run -d -p 3000:3000 --name aishort-offline rockben/chatgpt-shortcut:offline
```

ডিপ্লয়মেন্টের পরে দলের সদস্যরা `http://<সার্ভার IP>:3000` এ গিয়ে ব্যবহার করতে পারবেন।

`docker-compose` ব্যবহার:

```yml
services:
  aishort-offline:
    container_name: aishort-offline
    image: ghcr.io/rockbenben/chatgpt-shortcut:offline
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### সোর্স কোড বিল্ড

আপনার যদি প্রম্পট বিষয়বস্তু কাস্টমাইজ করতে বা কনফিগারেশন সংশোধন করতে হয়:

```bash
# অফলাইন ব্রাঞ্চ ক্লোন করুন
git clone -b offline https://github.com/rockbenben/ChatGPT-Shortcut.git
cd ChatGPT-Shortcut

# ডিপেন্ডেন্সি ইনস্টল করুন
yarn

# লোকাল ডেভেলপমেন্ট
yarn start

# একক ভাষার সংস্করণ বিল্ড (চীনা)
yarn build --locale zh-Hans

# সব ভাষা বিল্ড করুন
yarn build
```

বিল্ড আউটপুট `build/` ডিরেক্টরিতে থাকে, যেকোনো স্ট্যাটিক ফাইল সার্ভারে (Nginx, Apache, Caddy ইত্যাদি) ডিপ্লয় করা যায়।

### Nginx কনফিগারেশন উদাহরণ

```nginx
server {
    listen 3000;
    server_name _;
    root /path/to/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### প্ল্যাটফর্ম ডিপ্লয়মেন্ট

Vercel, Cloudflare Pages ইত্যাদি প্ল্যাটফর্মে ডিপ্লয় করার সময় `offline` ব্রাঞ্চ নির্বাচন করুন, অন্যান্য ধাপগুলি অনলাইন সংস্করণের মতোই, বিস্তারিত জানতে [প্রজেক্ট ডিপ্লয়মেন্ট](../deploy) দেখুন।

## ডেটা ইম্পোর্ট-এক্সপোর্ট

### এক্সপোর্ট

"সেটিংস > ডেটা এক্সপোর্ট" এ যান, ব্যক্তিগত পছন্দ এবং কাস্টম প্রম্পট JSON ফাইল হিসেবে এক্সপোর্ট করুন।

### ইম্পোর্ট

নিম্নলিখিত ফরম্যাটের JSON ফাইল ইম্পোর্ট সমর্থন করে:

- **অফলাইন সংস্করণ থেকে এক্সপোর্ট করা ফাইল**: পছন্দ, প্রম্পট, সাজানো এবং ট্যাগ সম্পূর্ণ পুনরুদ্ধার
- **অনলাইন সংস্করণ থেকে এক্সপোর্ট করা ফাইল**: স্বয়ংক্রিয়ভাবে সামঞ্জস্যপূর্ণ
  - ব্যবহারকারী প্রম্পট → স্থানীয়ে মার্জ (শিরোনাম অনুযায়ী ডুপ্লিকেট সরানো)
  - নির্বাচিত পছন্দ (card) → স্থানীয় পছন্দে মার্জ
  - কমিউনিটি পছন্দ (community) → স্বয়ংক্রিয়ভাবে স্থানীয় কাস্টম প্রম্পটে রূপান্তর
  - MySpace সাজানো → স্থানীয়ে পুনরুদ্ধার
  - কাস্টম ট্যাগ → যোগ করে মার্জ (বিদ্যমানগুলি ওভাররাইট করে না)

### অনলাইন সংস্করণ থেকে মাইগ্রেশন

1. অনলাইন সংস্করণ (aishort.top) এর "আমার অ্যাকাউন্ট" পৃষ্ঠা থেকে ডেটা এক্সপোর্ট করুন
2. অফলাইন সংস্করণের "সেটিংস" পৃষ্ঠায় সেই JSON ফাইলটি ইম্পোর্ট করুন
3. কমিউনিটি পছন্দ স্বয়ংক্রিয়ভাবে স্থানীয় প্রম্পটে রূপান্তরিত হবে, নির্বাচিত পছন্দ স্বাভাবিকভাবে সিঙ্ক হবে

## সাধারণ প্রশ্ন

### ডিপ্লয়মেন্টের পরে দল কীভাবে ব্যবহার করবে?

অ্যাডমিন ইন্ট্রানেট সার্ভারে ডিপ্লয় করার পরে, দলের সদস্যদের অ্যাক্সেস ঠিকানা (যেমন `http://192.168.1.100:3000`) জানিয়ে দিন। প্রত্যেকে ব্রাউজার দিয়ে খুলুন, ইনস্টলের প্রয়োজন নেই, নিবন্ধনের প্রয়োজন নেই।

### প্রত্যেকের ডেটা কি একে অপরকে প্রভাবিত করবে?

না। প্রত্যেকের পছন্দ এবং কাস্টম প্রম্পট তাদের নিজস্ব ব্রাউজারের localStorage-এ সংরক্ষিত, সম্পূর্ণ স্বতন্ত্র। সার্ভারে শুধুমাত্র শেয়ার করা প্রম্পট লাইব্রেরি (শুধুমাত্র পঠনযোগ্য) আছে।

### ডেটা কি হারিয়ে যেতে পারে?

নিম্নলিখিত অপারেশন ব্যক্তিগত ডেটা হারানোর কারণ হতে পারে:

- ব্রাউজার ডেটা/ক্যাশ মুছে ফেলা
- প্রাইভেট/ইনকগনিটো মোড ব্যবহার করা
- কম্পিউটার বা ব্রাউজার পরিবর্তন করা

গুরুত্বপূর্ণ ডেটা নিয়মিত "সেটিংস > ডেটা এক্সপোর্ট" এর মাধ্যমে JSON ফাইলে ব্যাকআপ নেওয়ার পরামর্শ দেওয়া হয়।

### দলের মধ্যে কাস্টম প্রম্পট শেয়ার করা কি সম্ভব?

হ্যাঁ। একজন ব্যক্তি JSON ফাইল এক্সপোর্ট করুন, তারপর অন্য সদস্যরা "সেটিংস > ডেটা ইম্পোর্ট" এ ইম্পোর্ট করুন। ইম্পোর্টের সময় স্বয়ংক্রিয়ভাবে ডুপ্লিকেট সরানো হয়।

### প্রম্পট লাইব্রেরি কীভাবে আপডেট করবেন?

প্রম্পট লাইব্রেরি হলো বিল্ড সময়ে প্যাক করা স্ট্যাটিক ডেটা। আপডেট পদ্ধতি:

1. অ্যাডমিন সর্বশেষ `offline` ব্রাঞ্চ কোড পুল করুন
2. পুনরায় বিল্ড এবং ডিপ্লয় করুন (অথবা সর্বশেষ Docker ইমেজ পুল করুন)
3. দলের সদস্যরা ব্রাউজার পৃষ্ঠা রিফ্রেশ করলে নতুন বিষয়বস্তু দেখা যাবে (ব্যক্তিগত ডেটা প্রভাবিত হবে না)

### অফলাইন সংস্করণের ডেটা ফরম্যাট কি অনলাইন সংস্করণের সাথে সামঞ্জস্যপূর্ণ?

সামঞ্জস্যপূর্ণ। এক্সপোর্ট করা JSON ফরম্যাট একই, দুটি সংস্করণের মধ্যে পারস্পরিক ইম্পোর্ট-এক্সপোর্ট করা যায়। প্রম্পট ID দুটি সংস্করণে ভিন্ন (অনলাইন সংস্করণ সার্ভার ID ব্যবহার করে, অফলাইন সংস্করণ টাইমস্ট্যাম্প ID ব্যবহার করে), কিন্তু ইম্পোর্টের সময় শিরোনাম অনুযায়ী ডুপ্লিকেট সরানো হয়, কোনো দ্বন্দ্ব হয় না।
