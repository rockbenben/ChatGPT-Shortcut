# Развертывание проекта

> **Для кого это руководство**: для разработчиков, которые хотят самостоятельно разместить или настроить AiShort. Обычным пользователям достаточно просто использовать [aishort.top](https://www.aishort.top) — читать это необязательно.

## Выберите модель развертывания

Выберите модель, соответствующую вашим потребностям:

| Модель                            | Бэкенд                                              | Примечания                                                                                                                                                                             |
| --------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Стандартная** (по умолчанию)    | Использует официальный общий бэкенд                 | После форка можно настраивать название сайта, описание, подсказки и т. д. (см. [Настройка](./deploy/configuration)); вход, избранное, сообщество и синхронизация работают сразу |
| **Офлайн-версия**                 | Без бэкенда, данные хранятся локально в браузере    | Изолированные корпоративные или государственные сети; аккаунт не требуется                                                                                                             |
| **Полностью автономный бэкенд**   | Ваш собственный независимый бэкенд                  | Когда необходима независимая система аккаунтов, полное владение данными и закрытое сообщество                                                                                          |

Первые две модели описаны в этом руководстве. Для третьей, поскольку бэкенд-сервис не является открытым исходным кодом, пожалуйста, [напишите разработчику](mailto:qingwhat@gmail.com), кратко описав ваш сценарий использования и масштаб, чтобы получить план развертывания и поддержку.

## Документация по развертыванию

Процесс развертывания разбит на следующие разделы — обращайтесь к нужному по необходимости:

- **[Стандартное развертывание](./deploy/standard)** — использует официальный общий бэкенд, поддерживает локальную сборку, Vercel, Cloudflare Pages и Docker.
- **[Офлайн-версия (корпоративная сеть)](./deploy/offline)** — офлайн-решение для изолированных сред, таких как корпоративные или государственные интрасети, без бэкенда и аккаунтов.
- **[Настройка и персонализация](./deploy/configuration)** — изменение заголовка сайта, описания, подсказок, а также подключение пользовательского бэкенда.
- **[Включение синхронизации обновлений](./deploy/sync-updates)** — пусть ваш форк автоматически следует за апстримом, чтобы не отставать по функциональности.
