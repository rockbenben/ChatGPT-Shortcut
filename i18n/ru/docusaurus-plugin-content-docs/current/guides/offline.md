# Версия для офлайн-развертывания

**Применимые сценарии**: Корпоративные интрасети, государственные сети, секретные среды, кампусные сети и другие сценарии, где **доступ к внешней сети недоступен или неудобен**.

Офлайн-версия AI Short не требует бэкенд-сервера и регистрации пользователей. Все данные хранятся локально в браузере. После развертывания она может использоваться командами напрямую во внутренней сети.

## Использование в команде

Офлайн-версия представляет собой чисто статический сайт. После развертывания на сервере интрасети членам команды достаточно открыть адрес интрасети в браузере:

1. **Администратор** развертывает офлайн-версию на сервере интрасети (например, `http://192.168.1.100:3000`)
2. **Члены команды** открывают адрес в браузере для просмотра, поиска и копирования промптов
3. **Избранное и пользовательские промпты каждого человека сохраняются в его собственном браузере**, независимо друг от друга
4. Регистрация не требуется, установка программного обеспечения не нужна, готово к использованию сразу

```
Сервер интрасети (офлайн-версия развернута)
   ├── Данные библиотеки промптов (общие для всех, из статического JSON)
   │
   ├── Браузер пользователя A → localStorage (избранное и пользовательские промпты A)
   ├── Браузер пользователя B → localStorage (избранное и пользовательские промпты B)
   └── Браузер пользователя C → localStorage (избранное и пользовательские промпты C)
```

:::tip Примечание
Содержимое библиотеки промптов (отобранные промпты) — это статические данные, упакованные во время сборки, и все пользователи видят одинаковый контент. Избранное, пользовательские промпты, сортировка и теги каждого пользователя сохраняются в localStorage его собственного браузера, полностью независимо друг от друга.
:::

## Хранение данных

Данные каждого пользователя сохраняются в localStorage **его собственного браузера**, независимо от сервера:

| Данные | Ключ хранения | Описание |
| ---- | ------ | ---- |
| Список избранного | `local_favorites` | Массив ID избранных промптов |
| Пользовательские промпты | `local_user_prompts` | Данные промптов, созданных пользователем |
| Порядок сортировки | `local_myspace_order` | Сортировка карточек в Моей Коллекции |
| Пользовательские теги | `local_custom_tags` | Определения и назначения тегов |

:::caution Внимание
- Локальное хранилище браузера имеет ограничение емкости примерно 5 МБ, что более чем достаточно для ежедневного использования.
- Очистка данных браузера приведет к потере персональных данных. Рекомендуется регулярно создавать резервные копии через «Настройки > Экспорт данных».
- После смены компьютера или браузера данные необходимо импортировать заново.
:::

## Развертывание

Офлайн-версия основана на ветке `offline`. После того как администратор завершит развертывание, члены команды могут использовать её без каких-либо дополнительных действий.

### Развертывание Docker (Рекомендуется)

Самый простой метод развертывания — одна команда для запуска на сервере интрасети:

```bash
# Использовать предварительно собранный офлайн-образ
docker run -d -p 3000:3000 --name aishort-offline ghcr.io/rockbenben/chatgpt-shortcut:offline

# Или использовать Docker Hub
docker run -d -p 3000:3000 --name aishort-offline rockben/chatgpt-shortcut:offline
```

После развертывания члены команды могут получить доступ по адресу `http://<IP-сервера>:3000`.

Использование `docker-compose`:

```yml
services:
  aishort-offline:
    container_name: aishort-offline
    image: ghcr.io/rockbenben/chatgpt-shortcut:offline
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### Сборка из исходного кода

Если вам нужно настроить содержимое промптов или изменить конфигурации:

```bash
# Клонировать офлайн-ветку
git clone -b offline https://github.com/rockbenben/ChatGPT-Shortcut.git
cd ChatGPT-Shortcut

# Установить зависимости
yarn

# Локальная разработка
yarn start

# Собрать одноязычную версию (китайский)
yarn build --locale zh-Hans

# Собрать все языки
yarn build
```

Результат сборки находится в каталоге `build/` и может быть развернут на любом сервере статических файлов (Nginx, Apache, Caddy и т.д.).

### Пример конфигурации Nginx

```nginx
server {
    listen 3000;
    server_name _;
    root /path/to/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### Развертывание на платформах

При развертывании на таких платформах, как Vercel или Cloudflare Pages, выберите ветку `offline`. Все остальные шаги идентичны онлайн-версии. Подробности см. в [Развертывание проекта](../deploy).

## Импорт и экспорт данных

### Экспорт

Перейдите в «Настройки > Экспорт данных», чтобы экспортировать ваше личное избранное и пользовательские промпты в файл JSON.

### Импорт

Для импорта поддерживаются следующие форматы файлов JSON:

- **Файлы, экспортированные из офлайн-версии**: Полное восстановление избранного, промптов, сортировки и тегов
- **Файлы, экспортированные из онлайн-версии**: Автоматическая обработка для совместимости
  - Пользовательские промпты → Объединяются в локальное хранилище (дедупликация по заголовку)
  - Избранные подборки (card) → Объединяются в локальное избранное
  - Избранное сообщества (community) → Автоматически конвертируется в локальные пользовательские промпты
  - Сортировка MySpace → Восстанавливается в локальном хранилище
  - Пользовательские теги → Добавляются и объединяются (не перезаписывают существующие)

### Миграция с онлайн-версии

1. Экспортируйте данные со страницы «Мой аккаунт» онлайн-версии (aishort.top)
2. Импортируйте файл JSON на странице «Настройки» офлайн-версии
3. Избранное сообщества автоматически конвертируется в локальные промпты, а избранные подборки синхронизируются нормально

## Часто задаваемые вопросы

### Как команда использует после развертывания?

После того как администратор развернет на сервере интрасети, просто поделитесь URL доступа (например, `http://192.168.1.100:3000`) с членами команды. Каждый открывает его в браузере — установка не нужна, регистрация не требуется.

### Влияют ли данные разных пользователей друг на друга?

Нет. Избранное и пользовательские промпты каждого человека сохраняются в localStorage его собственного браузера, полностью независимо. На сервере размещена только общая библиотека промптов (только для чтения).

### Могут ли данные быть потеряны?

Следующие действия приведут к потере персональных данных:

- Очистка данных/кэша браузера
- Просмотр в приватном/инкогнито режиме
- Смена компьютера или браузера

Рекомендуется регулярно создавать резервные копии важных данных через «Настройки > Экспорт данных» в файл JSON.

### Можно ли делиться пользовательскими промптами между членами команды?

Да. Один человек экспортирует файл JSON, а другие члены импортируют его через «Настройки > Импорт данных». Дубликаты автоматически удаляются при импорте.

### Как обновить библиотеку промптов?

Библиотека промптов — это статические данные, упакованные во время сборки. Для обновления:

1. Администратор получает последний код ветки `offline`
2. Пересобирает и повторно развертывает (или загружает последний образ Docker)
3. Члены команды обновляют страницу браузера, чтобы увидеть новый контент (персональные данные не затрагиваются)

### Совместим ли формат данных офлайн-версии с онлайн-версией?

Да. Экспортированный формат JSON идентичен и может импортироваться между двумя версиями. ID промптов различаются между версиями (онлайн-версия использует серверные ID, офлайн-версия использует ID на основе временных меток), но дедупликация выполняется по заголовку при импорте, поэтому конфликтов нет.
