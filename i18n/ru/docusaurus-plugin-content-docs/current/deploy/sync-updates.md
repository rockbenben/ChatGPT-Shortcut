# Включение синхронизации обновлений

После развертывания одним кликом на Vercel может постоянно отображаться уведомление о доступном обновлении — это происходит потому, что Vercel создаёт новый проект, а не форк, и поэтому не может обнаруживать обновления из источника. Чтобы исправить это:

1. Удалите исходный репозиторий
2. Используйте кнопку **Fork** в правом верхнем углу страницы, чтобы форкнуть этот проект
3. На [странице нового проекта Vercel](https://vercel.com/new) заново импортируйте форкнутый репозиторий через Import Git Repository и разверните

## Включить автоматическое обновление

> Если вы столкнулись с ошибкой Upstream Sync, выполните Sync Fork вручную один раз!

После форка вручную включите Workflows на странице Actions и запустите действие Upstream Sync один раз. После включения проект будет синхронизироваться автоматически каждый день:

## Ручное обновление

Хотите обновить немедленно вручную? См. [документацию GitHub по синхронизации форков](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/syncing-a-fork).

Вы также можете поставить звезду / подписаться на этот проект, чтобы получать уведомления о новых функциях.
