# 프로젝트 배포

> **대상 독자**: AiShort를 직접 호스팅하거나 커스터마이징하려는 개발자를 위한 가이드입니다. 일반 사용자는 [aishort.top](https://www.aishort.top)을 그대로 사용하면 됩니다 — 이 문서를 읽을 필요가 없습니다.

## 배포 모델 선택

필요에 맞는 모델을 선택하세요:

| 모델                        | 백엔드                              | 비고                                                                                                    |
| --------------------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------------- |
| **Standard** (기본값)       | 공식 공유 백엔드 재사용             | 포크 후 사이트 이름, 설명, 프롬프트 등을 커스터마이징할 수 있습니다 ([구성 및 커스터마이징](./deploy/configuration) 참조); 로그인, 즐겨찾기, 커뮤니티, 동기화가 기본 제공됩니다 |
| **오프라인 버전**           | 백엔드 없음, 데이터를 브라우저에 로컬 저장 | 망 분리된 기업 또는 정부 네트워크용; 계정 불필요                                                        |
| **완전 자체 호스팅 백엔드** | 독립적인 자체 백엔드                | 독립 계정 시스템, 완전한 데이터 소유권, 프라이빗 커뮤니티가 필요한 경우                                |

처음 두 가지는 이 가이드에서 설명합니다. 세 번째의 경우 백엔드 서비스가 오픈 소스가 아니므로, 사용 사례와 규모를 간략히 설명하여 [개발자에게 이메일](mailto:qingwhat@gmail.com)을 보내 배포 계획 및 지원을 받으세요.

## 배포 문서

배포 흐름에 따라 다음 문서로 나뉘어 있으니 필요에 따라 참조하세요:

- **[Standard 배포](./deploy/standard)** — 공식 공유 백엔드를 재사용하며, 로컬 빌드, Vercel, Cloudflare Pages, Docker 네 가지 방법을 지원합니다.
- **[오프라인 버전(사내 인트라넷)](./deploy/offline)** — 기업 인트라넷, 정부 네트워크 등 망 분리된 환경을 위한 오프라인 솔루션으로, 백엔드와 계정이 필요 없습니다.
- **[구성 및 커스터마이징](./deploy/configuration)** — 사이트 제목, 설명, 프롬프트 수정 및 커스텀 백엔드 연동.
- **[포크 동기화 유지](./deploy/sync-updates)** — 포크가 업스트림을 자동으로 따라가도록 하여 기능이 뒤처지지 않게 합니다.
