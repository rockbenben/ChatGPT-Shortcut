# 오프라인 배포 버전

**적용 시나리오**: 기업 인트라넷, 정부 네트워크, 보안 환경, 캠퍼스 네트워크 등 **외부 네트워크에 접속할 수 없거나 접속이 불편한** 시나리오.

AI Short 오프라인 버전은 백엔드 서버와 사용자 등록이 필요 없으며, 모든 데이터는 브라우저 로컬에 저장됩니다. 배포 후 인트라넷에서 팀이 바로 사용할 수 있습니다.

## 팀 사용 방법

오프라인 버전은 순수 정적 웹사이트입니다. 인트라넷 서버에 배포한 후 팀원은 브라우저로 해당 인트라넷 주소에 접속하기만 하면 사용할 수 있습니다:

1. **관리자**가 오프라인 버전을 인트라넷 서버에 배포 (예: `http://192.168.1.100:3000`)
2. **팀원**이 브라우저로 해당 주소를 열면 프롬프트 탐색, 검색, 복사 가능
3. 각 개인의 **즐겨찾기와 커스텀 프롬프트는 자신의 브라우저에 저장**되며 서로 영향을 주지 않음
4. 계정 등록 불필요, 소프트웨어 설치 불필요, 열기만 하면 바로 사용

```
인트라넷 서버 (오프라인 버전 배포)
   ├── 프롬프트 라이브러리 데이터 (모든 사용자 공유, 정적 JSON에서 제공)
   │
   ├── 사용자 A의 브라우저 → localStorage (A의 즐겨찾기 및 커스텀 프롬프트)
   ├── 사용자 B의 브라우저 → localStorage (B의 즐겨찾기 및 커스텀 프롬프트)
   └── 사용자 C의 브라우저 → localStorage (C의 즐겨찾기 및 커스텀 프롬프트)
```

:::tip 팁
프롬프트 라이브러리 콘텐츠(엄선된 프롬프트)는 빌드 시 패키징된 정적 데이터로, 모든 사용자에게 동일한 내용이 표시됩니다. 각 사용자의 즐겨찾기, 커스텀 프롬프트, 정렬 순서, 태그는 각자 브라우저의 localStorage에 저장되며 서로 독립적입니다.
:::

## 온라인 버전과의 차이

| 기능 | 온라인 버전 | 오프라인 버전 |
| ---- | ------ | ------ |
| 프롬프트 탐색/검색/필터 | ✅ | ✅ |
| 프롬프트 복사 | ✅ | ✅ |
| 즐겨찾기 관리 | 서버 저장 | 브라우저 로컬 저장 |
| 커스텀 프롬프트 | 서버 저장 | 브라우저 로컬 저장 |
| 마이 컬렉션 (드래그 정렬, 태그) | ✅ | ✅ |
| 다국어 지원 (18개 언어) | ✅ | ✅ |
| 데이터 가져오기/내보내기 | ✅ | ✅ (형식 호환) |
| 프롬프트 상세 페이지 | ✅ | ✅ (정적 데이터, 댓글 없음) |
| 사용자 등록/로그인 | ✅ | ❌ (계정 불필요) |
| 커뮤니티 프롬프트 목록/투표 | ✅ | ❌ |
| 댓글 피드백 | ✅ | ❌ |

## 데이터 저장

각 사용자의 데이터는 **자신의 브라우저** localStorage에 저장되며 서버와 무관합니다:

| 데이터 | 저장 키 | 설명 |
| ---- | ------ | ---- |
| 즐겨찾기 목록 | `local_favorites` | 즐겨찾기한 프롬프트 ID 배열 |
| 커스텀 프롬프트 | `local_user_prompts` | 사용자가 생성한 프롬프트 데이터 |
| 정렬 순서 | `local_myspace_order` | 마이 컬렉션의 카드 정렬 순서 |
| 커스텀 태그 | `local_custom_tags` | 태그 정의 및 할당 관계 |

:::caution 주의
- 브라우저 로컬 저장소는 약 5MB의 용량 제한이 있지만 일상적인 사용에는 충분합니다.
- 브라우저 데이터를 삭제하면 개인 데이터가 손실됩니다. 정기적으로 "설정 > 데이터 내보내기"를 통해 백업하는 것을 권장합니다.
- 컴퓨터나 브라우저를 변경하면 데이터를 다시 가져와야 합니다.
:::

## 배포

오프라인 버전은 `offline` 브랜치를 기반으로 합니다. 관리자가 한 번 배포하면 팀원은 별도의 작업 없이 사용할 수 있습니다.

### Docker 배포 (권장)

가장 간단한 배포 방법으로, 한 줄의 명령으로 인트라넷 서버에서 실행할 수 있습니다:

```bash
# 사전 빌드된 오프라인 버전 이미지 사용
docker run -d -p 3000:3000 --name aishort-offline ghcr.io/rockbenben/chatgpt-shortcut:offline

# 또는 Docker Hub 사용
docker run -d -p 3000:3000 --name aishort-offline rockben/chatgpt-shortcut:offline
```

배포 후 팀원은 `http://<서버IP>:3000`에 접속하면 바로 사용할 수 있습니다.

`docker-compose` 사용 시:

```yml
services:
  aishort-offline:
    container_name: aishort-offline
    image: ghcr.io/rockbenben/chatgpt-shortcut:offline
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### 소스 코드 빌드

프롬프트 콘텐츠를 커스터마이징하거나 설정을 변경해야 하는 경우:

```bash
# 오프라인 버전 브랜치 클론
git clone -b offline https://github.com/rockbenben/ChatGPT-Shortcut.git
cd ChatGPT-Shortcut

# 의존성 설치
yarn

# 로컬 개발
yarn start

# 단일 언어 버전 (중국어) 빌드
yarn build --locale zh-Hans

# 전체 언어 빌드
yarn build
```

빌드 결과물은 `build/` 디렉터리에 생성되며, 임의의 정적 파일 서버(Nginx, Apache, Caddy 등)에 배포할 수 있습니다.

### Nginx 구성 예시

```nginx
server {
    listen 3000;
    server_name _;
    root /path/to/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 플랫폼 배포

Vercel, Cloudflare Pages 등의 플랫폼에 배포할 때 `offline` 브랜치를 선택하기만 하면 되며, 나머지 절차는 온라인 버전과 동일합니다. 자세한 내용은 [프로젝트 배포](../deploy)를 참조하세요.

## 데이터 가져오기/내보내기

### 내보내기

"설정 > 데이터 내보내기"에서 개인 즐겨찾기와 커스텀 프롬프트를 JSON 파일로 내보냅니다.

### 가져오기

다음 형식의 JSON 파일 가져오기를 지원합니다:

- **오프라인 버전에서 내보낸 파일**: 즐겨찾기, 프롬프트, 정렬 순서, 태그를 완전히 복원
- **온라인 버전에서 내보낸 파일**: 자동 호환 처리
  - 사용자 프롬프트 → 로컬에 병합 (제목으로 중복 제거)
  - 엄선 즐겨찾기 (card) → 로컬 즐겨찾기에 병합
  - 커뮤니티 즐겨찾기 (community) → 자동으로 로컬 커스텀 프롬프트로 변환
  - MySpace 정렬 순서 → 로컬에 복원
  - 커스텀 태그 → 추가 병합 (기존 항목 덮어쓰지 않음)

### 온라인 버전에서 마이그레이션

1. 온라인 버전(aishort.top)의 "내 계정" 페이지에서 데이터 내보내기
2. 오프라인 버전의 "설정" 페이지에서 해당 JSON 파일 가져오기
3. 커뮤니티 즐겨찾기는 자동으로 로컬 프롬프트로 변환되고, 엄선 즐겨찾기는 정상적으로 동기화

## 자주 묻는 질문

### 배포 후 팀은 어떻게 사용하나요?

관리자가 인트라넷 서버에 배포한 후 접속 주소(예: `http://192.168.1.100:3000`)를 팀원에게 알려주면 됩니다. 각자 브라우저로 열기만 하면 되며, 설치 불필요, 등록 불필요입니다.

### 각 개인의 데이터가 서로 영향을 주나요?

아닙니다. 각 개인의 즐겨찾기와 커스텀 프롬프트는 각자 브라우저의 localStorage에 저장되어 완전히 독립적입니다. 서버에는 공유 프롬프트 라이브러리(읽기 전용)만 있습니다.

### 데이터가 손실될 수 있나요?

다음 작업은 개인 데이터 손실을 유발할 수 있습니다:

- 브라우저 데이터/캐시 삭제
- 프라이빗/시크릿 모드로 탐색
- 컴퓨터 또는 브라우저 변경

중요한 데이터는 정기적으로 "설정 > 데이터 내보내기"를 통해 JSON 파일로 백업하는 것을 권장합니다.

### 팀 간에 커스텀 프롬프트를 공유할 수 있나요?

가능합니다. 한 사람이 JSON 파일을 내보낸 후 다른 팀원이 "설정 > 데이터 가져오기"에서 가져오면 됩니다. 가져올 때 자동으로 중복이 제거됩니다.

### 프롬프트 라이브러리를 업데이트하려면?

프롬프트 라이브러리는 빌드 시 패키징된 정적 데이터입니다. 업데이트 방법:

1. 관리자가 최신 `offline` 브랜치 코드를 풀
2. 다시 빌드하여 배포 (또는 최신 Docker 이미지 풀)
3. 팀원이 브라우저 페이지를 새로고침하면 새 콘텐츠가 표시됩니다 (개인 데이터는 영향 없음)

### 오프라인 버전의 데이터 형식은 온라인 버전과 호환되나요?

호환됩니다. 내보낸 JSON 형식이 동일하여 두 버전 간에 상호 가져오기가 가능합니다. 프롬프트 ID는 두 버전에서 다릅니다 (온라인 버전은 서버 ID, 오프라인 버전은 타임스탬프 ID 사용). 하지만 가져올 때 제목으로 중복이 제거되므로 충돌하지 않습니다.
