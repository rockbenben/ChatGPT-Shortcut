# 커스텀 프롬프트

로그인 후에는 나만의 프롬프트를 생성하여 계정에 저장하고 언제든지 호출할 수 있습니다. 커스텀 프롬프트는 "마이 컬렉션" 뷰에 표시됩니다.

## 프롬프트 생성

1. 오른쪽 상단의 "추가" 버튼 클릭
2. 팝업으로 열리는 "프롬프트 생성" 창에서 폼을 작성합니다:
   - **프롬프트 이름** (필수): 프롬프트의 이름을 지정합니다
   - **프롬프트 내용** (필수): 프롬프트 본문
   - **역할·용도** (선택): 프롬프트가 수행하는 작업을 간략히 설명
   - **비고** (선택): 출처, 다른 언어 버전 또는 추가 설명
3. 하단의 "이 프롬프트를 공개 페이지에 공유하시겠습니까?" 스위치는 기본적으로 켜져 있으며 끄면 자신만 볼 수 있습니다
4. "프롬프트 생성"을 클릭하여 제출

![프롬프트 생성 팝업](/img/docs/user-prompts-create.png)

> 💡 **작성 예시** — 「회의록 요약 어시스턴트」 프롬프트:
> - **프롬프트 이름**: 회의록 요약 어시스턴트
> - **프롬프트 내용**: 당신은 회의록 정리 전문가입니다. 다음 회의 내용을 바탕으로 핵심 논의 사항, 결정 사항, 액션 아이템(담당자·기한 포함)을 구분하여 한 페이지 분량으로 요약해 주십시오: [회의 내용 또는 녹취록]
> - **역할·용도**: 장황한 회의 기록을 팀과 상급자가 빠르게 파악할 수 있는 구조화된 회의록으로 정리
> - **비고**: 주간 회의 및 프로젝트 킥오프 후 사용

## 프롬프트 편집

마이 컬렉션 뷰에서 자신이 생성한 프롬프트 카드를 클릭하여 "프롬프트 편집" 창을 열면 다음을 수행할 수 있습니다:

- 이름, 내용, 역할·용도, 비고 수정
- 공유 상태 전환 (공개/비공개)
- "저장"을 클릭하여 업데이트

![프롬프트 편집 인터페이스](/img/docs/user-prompts-edit.png)

## 프롬프트 삭제

편집 창에서 "삭제"를 클릭하면 됩니다. 삭제 후에는 복구할 수 없으므로 신중하게 진행해 주십시오.

## 커뮤니티에 공유

생성 또는 편집 시 하단의 "이 프롬프트를 공개 페이지에 공유하시겠습니까?" 스위치가 가시성을 제어합니다:

- **켜기 (기본값)**: 프롬프트가 [커뮤니티 프롬프트](./community) 페이지에 표시되며 다른 사용자가 보고 수집할 수 있습니다
- **끄기**: 비공개 프롬프트로 자신만 볼 수 있습니다

공유 상태는 언제든지 전환할 수 있습니다.

## 백업 내보내기

1. "내 계정" → "데이터 관리 → 프롬프트 내보내기" 항목 찾기
2. "데이터 내보내기" 버튼 클릭
3. 시스템이 JSON 파일을 생성하여 자동으로 다운로드

내보내기 내용에는 다음이 포함됩니다:

- 프롬프트의 이름, 내용, 역할·용도, 비고
- 생성 및 업데이트 시간
- 공유 상태

데이터 손실을 방지하기 위해 정기적으로 백업하는 것이 좋습니다.

## 프롬프트 가져오기

JSON 파일에서 프롬프트와 즐겨찾기를 가져옵니다:

1. "내 계정" → "데이터 관리 → 프롬프트 가져오기" 항목 찾기
2. "데이터 가져오기" 버튼 클릭
3. JSON 파일 선택
4. 시스템이 자동으로 데이터를 병합합니다(제목 기준으로 중복 제거. 타인 ID의 프롬프트는 자동으로 비공개로 설정됨)

### 팀 협업

팀 내에서 프롬프트를 공유하는 권장 워크플로우:

1. **필터링 및 공유**: 내보낸 후 즐겨찾기 목록을 수동으로 삭제하거나 공유할 프롬프트만 필터링한 다음 팀원에게 파일을 전달하여 가져오게 합니다
2. **개인정보 보호**: 다른 사람이 공유한 프롬프트(현재 계정에 속하지 않는 ID)를 가져오면 자동으로 **비공개**로 설정되어 멤버 간 데이터가 서로 간섭하지 않습니다
