# 커스텀 프롬프트

로그인 후에는 나만의 프롬프트를 생성하여 계정에 저장하고 언제든지 호출할 수 있습니다. 커스텀 프롬프트는 "마이 컬렉션" 뷰에 표시됩니다.

## 프롬프트 생성

1. 로그인 후 페이지 상단의 "**프롬프트 만들기**" 버튼을 클릭합니다
2. "프롬프트 만들기" 대화상자에서 폼을 작성합니다:
   - **프롬프트 이름** (필수): 프롬프트의 이름을 지정합니다
   - **프롬프트 내용** (필수): 프롬프트 본문. 대괄호 `[...]` 는 표시(렌더링)될 때 플레이스홀더로 강조됩니다
   - **용도** (선택): 프롬프트가 수행하는 작업을 간략히 설명합니다
   - **비고** (선택): 출처, 다른 언어 버전 또는 추가 설명
3. 하단의 "**이 프롬프트를 공개 페이지에 공유하시겠습니까?**" 스위치는 기본적으로 켜져 있으며, 끄면 비공개로 유지됩니다
4. "프롬프트 만들기"를 클릭하여 제출합니다

![프롬프트 생성 팝업](/img/docs/user-prompts-create.png)

> 💡 **작성 예시** — 「스탠드업 업데이트 생성기」 프롬프트:
> - **프롬프트 이름**: 스탠드업 업데이트 생성기
> - **프롬프트 내용**: 당신은 간결한 커뮤니케이션 어시스턴트입니다. 다음 메모를 어제 한 일, 오늘 할 일, 방해 요인의 세 섹션으로 구성된 일일 스탠드업 업데이트로 바꿔 주세요. 각 항목은 짧고 결과 중심으로 작성해 주세요: [내 메모]
> - **용도**: 정리되지 않은 메모를 깔끔한 스탠드업 업데이트로 변환합니다
> - **비고**: 아침 스탠드업 전에 실행하세요

## 프롬프트 편집

마이 컬렉션 뷰에서 내가 만든 프롬프트 카드의 편집(연필) 버튼을 클릭하면 "프롬프트 편집" 대화상자가 열립니다. 다음을 수행할 수 있습니다:

- 이름, 내용, 용도, 비고 수정
- 공유 상태 전환 (공개 / 비공개)
- "저장"을 클릭하여 업데이트

![프롬프트 편집 인터페이스](/img/docs/user-prompts-edit.png)

## 프롬프트 삭제

편집 대화상자에서 "삭제"를 클릭합니다. 삭제 후에는 복구할 수 없으므로 신중하게 진행해 주세요.

## 커뮤니티에 공유

생성 또는 편집 시 하단의 "이 프롬프트를 공개 페이지에 공유하시겠습니까?" 스위치가 공개 여부를 제어합니다:

- **켜기 (기본값)**: 프롬프트가 [커뮤니티 프롬프트](./community) 페이지에 표시되며 다른 사용자가 보고 즐겨찾기에 담을 수 있습니다
- **끄기**: 비공개 프롬프트로 본인만 볼 수 있습니다

공유 상태는 언제든지 전환할 수 있습니다.

## 프롬프트 내보내기

1. "내 계정"으로 이동하여 "데이터 관리 → **프롬프트 내보내기**" 항목을 찾습니다
2. "데이터 내보내기" 버튼을 클릭합니다
3. 시스템이 JSON 파일을 생성하여 자동으로 다운로드합니다

내보내기 내용에는 다음이 포함됩니다:

- 프롬프트의 이름, 내용, 용도, 비고
- 공유 상태
- 즐겨찾기, 즐겨찾기 정렬 순서, 사용자 지정 태그

데이터 손실을 방지하기 위해 정기적으로 내보내기하는 것을 권장합니다.

## 프롬프트 가져오기

JSON 파일에서 프롬프트와 즐겨찾기를 가져옵니다:

1. "내 계정"으로 이동하여 "데이터 관리 → **프롬프트 가져오기**" 항목을 찾습니다
2. "데이터 가져오기" 버튼을 클릭합니다
3. JSON 파일을 선택합니다
4. 시스템이 자동으로 데이터를 병합합니다(제목 기준으로 중복 제거. 타인 계정 ID에 속하는 프롬프트는 자동으로 비공개로 설정됩니다)

### 팀 협업

팀 내에서 프롬프트를 공유하는 권장 워크플로우:

1. **필터링 및 공유**: 내보낸 후 즐겨찾기 목록을 수동으로 제거하거나 공유할 프롬프트만 추려내어 팀원에게 파일을 전달하고 가져오게 합니다
2. **개인정보 보호**: 다른 사람의 프롬프트(현재 계정에 속하지 않는 ID)를 가져오면 자동으로 **비공개**로 설정되어 멤버 간 데이터가 서로 간섭하지 않습니다
