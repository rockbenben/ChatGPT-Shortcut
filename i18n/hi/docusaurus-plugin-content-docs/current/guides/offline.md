# ऑफ़लाइन डिप्लॉयमेंट संस्करण

**उपयुक्त परिदृश्य**: एंटरप्राइज़ इंट्रानेट, सरकारी नेटवर्क, गोपनीय वातावरण, कैंपस नेटवर्क आदि **जहां बाहरी नेटवर्क तक पहुंच संभव नहीं या असुविधाजनक है**।

AI Short ऑफ़लाइन संस्करण को किसी बैकएंड सर्वर या उपयोगकर्ता पंजीकरण की आवश्यकता नहीं है, सभी डेटा ब्राउज़र में स्थानीय रूप से संग्रहीत होता है। डिप्लॉयमेंट के बाद टीम इंट्रानेट में सीधे उपयोग कर सकती है।

## टीम उपयोग विधि

ऑफ़लाइन संस्करण एक शुद्ध स्टैटिक वेबसाइट है, इंट्रानेट सर्वर पर डिप्लॉय करने के बाद, टीम के सदस्य बस ब्राउज़र से उस इंट्रानेट एड्रेस पर जाकर उपयोग कर सकते हैं:

1. **एडमिन** ऑफ़लाइन संस्करण को इंट्रानेट सर्वर पर डिप्लॉय करता है (जैसे `http://192.168.1.100:3000`)
2. **टीम के सदस्य** ब्राउज़र से उस एड्रेस को खोलते हैं, प्रॉम्प्ट ब्राउज़, खोज और कॉपी कर सकते हैं
3. प्रत्येक व्यक्ति का **पसंदीदा और कस्टम प्रॉम्प्ट अपने ब्राउज़र में सेव** होता है, एक दूसरे को प्रभावित नहीं करता
4. खाता पंजीकरण की आवश्यकता नहीं, कोई सॉफ्टवेयर इंस्टॉल करने की आवश्यकता नहीं, खोलें और उपयोग करें

```
इंट्रानेट सर्वर (ऑफ़लाइन संस्करण डिप्लॉय)
   ├── प्रॉम्प्ट लाइब्रेरी डेटा (सभी के लिए साझा, स्टैटिक JSON से)
   │
   ├── उपयोगकर्ता A का ब्राउज़र → localStorage (A का पसंदीदा और कस्टम प्रॉम्प्ट)
   ├── उपयोगकर्ता B का ब्राउज़र → localStorage (B का पसंदीदा और कस्टम प्रॉम्प्ट)
   └── उपयोगकर्ता C का ब्राउज़र → localStorage (C का पसंदीदा और कस्टम प्रॉम्प्ट)
```

:::tip सुझाव
प्रॉम्प्ट लाइब्रेरी की सामग्री (चयनित प्रॉम्प्ट) बिल्ड समय पर पैक किया गया स्टैटिक डेटा है, सभी उपयोगकर्ता एक ही सामग्री देखते हैं। प्रत्येक उपयोगकर्ता का पसंदीदा, कस्टम प्रॉम्प्ट, क्रम और टैग उनके अपने ब्राउज़र के localStorage में सहेजे जाते हैं, जो एक दूसरे से स्वतंत्र हैं।
:::

## ऑनलाइन संस्करण से अंतर

| फ़ीचर | ऑनलाइन संस्करण | ऑफ़लाइन संस्करण |
| ---- | ------ | ------ |
| प्रॉम्प्ट ब्राउज़/खोज/फ़िल्टर | ✅ | ✅ |
| प्रॉम्प्ट कॉपी | ✅ | ✅ |
| पसंदीदा प्रबंधन | सर्वर स्टोरेज | ब्राउज़र लोकल स्टोरेज |
| कस्टम प्रॉम्प्ट | सर्वर स्टोरेज | ब्राउज़र लोकल स्टोरेज |
| मेरा संग्रह (ड्रैग सॉर्टिंग, टैग) | ✅ | ✅ |
| बहु-भाषा समर्थन (18 भाषाएं) | ✅ | ✅ |
| डेटा आयात/निर्यात | ✅ | ✅ (प्रारूप संगत) |
| प्रॉम्प्ट विवरण पृष्ठ | ✅ | ✅ (स्टैटिक डेटा, बिना टिप्पणी) |
| उपयोगकर्ता पंजीकरण/लॉगिन | ✅ | ❌ (खाते की आवश्यकता नहीं) |
| सामुदायिक प्रॉम्प्ट सूची/वोटिंग | ✅ | ❌ |
| टिप्पणी प्रतिक्रिया | ✅ | ❌ |

## डेटा स्टोरेज

प्रत्येक उपयोगकर्ता का डेटा **अपने ब्राउज़र** के localStorage में सहेजा जाता है, सर्वर से कोई संबंध नहीं:

| डेटा | स्टोरेज कुंजी | विवरण |
| ---- | ------ | ---- |
| पसंदीदा सूची | `local_favorites` | पसंदीदा प्रॉम्प्ट ID की सरणी |
| कस्टम प्रॉम्प्ट | `local_user_prompts` | उपयोगकर्ता द्वारा बनाए गए प्रॉम्प्ट डेटा |
| क्रम | `local_myspace_order` | मेरे संग्रह में कार्ड का क्रम |
| कस्टम टैग | `local_custom_tags` | टैग परिभाषा और असाइनमेंट संबंध |

:::caution ध्यान दें
- ब्राउज़र लोकल स्टोरेज की लगभग 5MB की क्षमता सीमा है, दैनिक उपयोग के लिए पर्याप्त है।
- ब्राउज़र डेटा साफ़ करने से व्यक्तिगत डेटा खो जाएगा, नियमित रूप से "सेटिंग > डेटा निर्यात" के माध्यम से बैकअप लेने की सिफारिश की जाती है।
- कंप्यूटर या ब्राउज़र बदलने के बाद डेटा को फिर से आयात करना होगा।
:::

## डिप्लॉयमेंट

ऑफ़लाइन संस्करण `offline` ब्रांच पर आधारित है। एडमिन द्वारा एक बार डिप्लॉयमेंट पूरा करने के बाद, टीम के सदस्यों को कोई कार्रवाई करने की आवश्यकता नहीं है।

### Docker डिप्लॉयमेंट (अनुशंसित)

सबसे सरल डिप्लॉयमेंट विधि, एक कमांड से इंट्रानेट सर्वर पर चला सकते हैं:

```bash
# पूर्व-निर्मित ऑफ़लाइन संस्करण इमेज का उपयोग करें
docker run -d -p 3000:3000 --name aishort-offline ghcr.io/rockbenben/chatgpt-shortcut:offline

# या Docker Hub का उपयोग करें
docker run -d -p 3000:3000 --name aishort-offline rockben/chatgpt-shortcut:offline
```

डिप्लॉयमेंट के बाद टीम के सदस्य `http://<सर्वर IP>:3000` पर जाकर उपयोग कर सकते हैं।

`docker-compose` का उपयोग:

```yml
services:
  aishort-offline:
    container_name: aishort-offline
    image: ghcr.io/rockbenben/chatgpt-shortcut:offline
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### सोर्स कोड बिल्ड

यदि आपको प्रॉम्प्ट सामग्री को अनुकूलित करना या कॉन्फ़िगरेशन संशोधित करना है:

```bash
# ऑफ़लाइन ब्रांच क्लोन करें
git clone -b offline https://github.com/rockbenben/ChatGPT-Shortcut.git
cd ChatGPT-Shortcut

# डिपेंडेंसी इंस्टॉल करें
yarn

# स्थानीय विकास
yarn start

# एकल भाषा संस्करण बिल्ड (चीनी)
yarn build --locale zh-Hans

# सभी भाषाएं बिल्ड करें
yarn build
```

बिल्ड आउटपुट `build/` डायरेक्टरी में होता है, जिसे किसी भी स्टैटिक फ़ाइल सर्वर (Nginx, Apache, Caddy आदि) पर डिप्लॉय किया जा सकता है।

### Nginx कॉन्फ़िगरेशन उदाहरण

```nginx
server {
    listen 3000;
    server_name _;
    root /path/to/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### प्लेटफ़ॉर्म डिप्लॉयमेंट

Vercel, Cloudflare Pages आदि प्लेटफ़ॉर्म पर डिप्लॉय करते समय `offline` ब्रांच चुनें, अन्य चरण ऑनलाइन संस्करण के समान हैं, विवरण के लिए [प्रोजेक्ट डिप्लॉयमेंट](../deploy) देखें।

## डेटा आयात-निर्यात

### निर्यात

"सेटिंग > डेटा निर्यात" पर जाएं, व्यक्तिगत पसंदीदा और कस्टम प्रॉम्प्ट को JSON फ़ाइल के रूप में निर्यात करें।

### आयात

निम्नलिखित प्रारूप की JSON फ़ाइलें आयात करने का समर्थन करता है:

- **ऑफ़लाइन संस्करण से निर्यात की गई फ़ाइल**: पसंदीदा, प्रॉम्प्ट, क्रम और टैग पूरी तरह से पुनर्स्थापित
- **ऑनलाइन संस्करण से निर्यात की गई फ़ाइल**: स्वचालित रूप से संगत
  - उपयोगकर्ता प्रॉम्प्ट → स्थानीय में मर्ज (शीर्षक के आधार पर डुप्लीकेट हटाना)
  - चयनित पसंदीदा (card) → स्थानीय पसंदीदा में मर्ज
  - सामुदायिक पसंदीदा (community) → स्वचालित रूप से स्थानीय कस्टम प्रॉम्प्ट में परिवर्तित
  - MySpace क्रम → स्थानीय में पुनर्स्थापित
  - कस्टम टैग → जोड़कर मर्ज (मौजूदा को ओवरराइट नहीं करता)

### ऑनलाइन संस्करण से माइग्रेशन

1. ऑनलाइन संस्करण (aishort.top) के "मेरा खाता" पृष्ठ से डेटा निर्यात करें
2. ऑफ़लाइन संस्करण के "सेटिंग" पृष्ठ पर उस JSON फ़ाइल को आयात करें
3. सामुदायिक पसंदीदा स्वचालित रूप से स्थानीय प्रॉम्प्ट में परिवर्तित हो जाएंगे, चयनित पसंदीदा सामान्य रूप से सिंक होंगे

## सामान्य प्रश्न

### डिप्लॉयमेंट के बाद टीम कैसे उपयोग करेगी?

एडमिन द्वारा इंट्रानेट सर्वर पर डिप्लॉय करने के बाद, टीम के सदस्यों को एक्सेस एड्रेस (जैसे `http://192.168.1.100:3000`) बता दें। हर कोई ब्राउज़र में खोले, इंस्टॉल करने की आवश्यकता नहीं, पंजीकरण की आवश्यकता नहीं।

### क्या हर किसी का डेटा एक दूसरे को प्रभावित करेगा?

नहीं। हर किसी का पसंदीदा और कस्टम प्रॉम्प्ट उनके अपने ब्राउज़र के localStorage में सहेजा जाता है, पूरी तरह से स्वतंत्र। सर्वर पर केवल साझा प्रॉम्प्ट लाइब्रेरी (केवल पढ़ने योग्य) है।

### क्या डेटा खो सकता है?

निम्नलिखित ऑपरेशन से व्यक्तिगत डेटा खो सकता है:

- ब्राउज़र डेटा/कैश साफ़ करना
- प्राइवेट/इनकॉग्निटो मोड का उपयोग करना
- कंप्यूटर या ब्राउज़र बदलना

महत्वपूर्ण डेटा को नियमित रूप से "सेटिंग > डेटा निर्यात" के माध्यम से JSON फ़ाइल में बैकअप लेने की सिफारिश की जाती है।

### क्या टीम के बीच कस्टम प्रॉम्प्ट साझा किए जा सकते हैं?

हां। एक व्यक्ति JSON फ़ाइल निर्यात करे, फिर अन्य सदस्य "सेटिंग > डेटा आयात" में आयात करें। आयात करते समय स्वचालित रूप से डुप्लीकेट हटा दिए जाते हैं।

### प्रॉम्प्ट लाइब्रेरी कैसे अपडेट करें?

प्रॉम्प्ट लाइब्रेरी बिल्ड समय पर पैक किया गया स्टैटिक डेटा है। अपडेट विधि:

1. एडमिन नवीनतम `offline` ब्रांच कोड पुल करे
2. पुनः बिल्ड और डिप्लॉय करें (या नवीनतम Docker इमेज पुल करें)
3. टीम के सदस्य ब्राउज़र पृष्ठ रिफ्रेश करें तो नई सामग्री दिखाई देगी (व्यक्तिगत डेटा प्रभावित नहीं होता)

### क्या ऑफ़लाइन संस्करण का डेटा प्रारूप ऑनलाइन संस्करण के साथ संगत है?

संगत है। निर्यात किया गया JSON प्रारूप समान है, दोनों संस्करणों के बीच आयात-निर्यात किया जा सकता है। प्रॉम्प्ट ID दोनों संस्करणों में अलग-अलग हैं (ऑनलाइन संस्करण सर्वर ID का उपयोग करता है, ऑफ़लाइन संस्करण टाइमस्टैम्प ID का उपयोग करता है), लेकिन आयात करते समय शीर्षक के आधार पर डुप्लीकेट हटाए जाते हैं, कोई विरोध नहीं होता।
