# オフラインデプロイ版

**適用シナリオ**：企業イントラネット、政府ネットワーク、機密環境、キャンパスネットワークなど、**外部ネットワークにアクセスできない、またはアクセスが不便な**シナリオ。

AI Short オフライン版は、バックエンドサーバーやユーザー登録が不要で、すべてのデータはブラウザにローカル保存されます。デプロイ後、イントラネット内でチームがすぐに使用できます。

## チームでの使い方

オフライン版は純粋な静的ウェブサイトです。イントラネットサーバーにデプロイした後、チームメンバーはブラウザでそのイントラネットアドレスにアクセスするだけで使用できます：

1. **管理者**がオフライン版をイントラネットサーバーにデプロイ（例：`http://192.168.1.100:3000`）
2. **チームメンバー**がブラウザでそのアドレスを開き、プロンプトの閲覧、検索、コピーが可能
3. 各個人の**コレクションとカスタムプロンプトは自分のブラウザに保存**され、互いに影響しません
4. アカウント登録不要、ソフトウェアのインストール不要、開くだけで使用可能

```
イントラネットサーバー（オフライン版デプロイ）
   ├── プロンプトライブラリデータ（全員共有、静的 JSON から）
   │
   ├── ユーザー A のブラウザ → localStorage（A のコレクションとカスタムプロンプト）
   ├── ユーザー B のブラウザ → localStorage（B のコレクションとカスタムプロンプト）
   └── ユーザー C のブラウザ → localStorage（C のコレクションとカスタムプロンプト）
```

:::tip ヒント
プロンプトライブラリのコンテンツ（厳選プロンプト）はビルド時にパッケージされた静的データであり、すべてのユーザーに同じ内容が表示されます。各ユーザーのコレクション、カスタムプロンプト、並び順、タグは各自のブラウザの localStorage に保存され、互いに独立しています。
:::

## オンライン版との違い

| 機能 | オンライン版 | オフライン版 |
| ---- | ------ | ------ |
| プロンプト閲覧/検索/フィルタ | ✅ | ✅ |
| プロンプトコピー | ✅ | ✅ |
| コレクション管理 | サーバー保存 | ブラウザローカル保存 |
| カスタムプロンプト | サーバー保存 | ブラウザローカル保存 |
| マイコレクション（ドラッグ並べ替え・タグ） | ✅ | ✅ |
| 多言語サポート（18言語） | ✅ | ✅ |
| データのインポート/エクスポート | ✅ | ✅（形式互換） |
| プロンプト詳細ページ | ✅ | ✅（静的データ、コメントなし） |
| ユーザー登録/ログイン | ✅ | ❌（アカウント不要） |
| コミュニティプロンプト一覧/投票 | ✅ | ❌ |
| コメント・フィードバック | ✅ | ❌ |

## データストレージ

各ユーザーのデータは**自分のブラウザ**の localStorage に保存され、サーバーとは無関係です：

| データ | ストレージキー | 説明 |
| ---- | ------ | ---- |
| コレクションリスト | `local_favorites` | コレクションしたプロンプト ID の配列 |
| カスタムプロンプト | `local_user_prompts` | ユーザーが作成したプロンプトデータ |
| 並び順 | `local_myspace_order` | マイコレクション内のカード並び順 |
| カスタムタグ | `local_custom_tags` | タグ定義と割り当て関係 |

:::caution 注意
- ブラウザのローカルストレージには約 5MB の容量制限がありますが、日常的な使用には十分です。
- ブラウザのデータを消去すると個人データが失われます。定期的に「設定 > データエクスポート」でバックアップすることをお勧めします。
- パソコンやブラウザを変更した場合は、データを再インポートする必要があります。
:::

## デプロイ

オフライン版は `offline` ブランチに基づいています。管理者が一度デプロイすれば、チームメンバーは何も操作する必要なく使用できます。

### Docker デプロイ（推奨）

最も簡単なデプロイ方法で、一行のコマンドでイントラネットサーバー上で実行できます：

```bash
# ビルド済みオフライン版イメージを使用
docker run -d -p 3000:3000 --name aishort-offline ghcr.io/rockbenben/chatgpt-shortcut:offline

# または Docker Hub を使用
docker run -d -p 3000:3000 --name aishort-offline rockben/chatgpt-shortcut:offline
```

デプロイ後、チームメンバーは `http://<サーバーIP>:3000` にアクセスするだけで使用できます。

`docker-compose` を使用する場合：

```yml
services:
  aishort-offline:
    container_name: aishort-offline
    image: ghcr.io/rockbenben/chatgpt-shortcut:offline
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### ソースコードビルド

プロンプトコンテンツのカスタマイズや設定の変更が必要な場合：

```bash
# オフライン版ブランチをクローン
git clone -b offline https://github.com/rockbenben/ChatGPT-Shortcut.git
cd ChatGPT-Shortcut

# 依存関係をインストール
yarn

# ローカル開発
yarn start

# 単一言語版（中国語）をビルド
yarn build --locale zh-Hans

# 全言語をビルド
yarn build
```

ビルド成果物は `build/` ディレクトリに生成され、任意の静的ファイルサーバー（Nginx、Apache、Caddy など）にデプロイできます。

### Nginx 設定例

```nginx
server {
    listen 3000;
    server_name _;
    root /path/to/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### プラットフォームデプロイ

Vercel、Cloudflare Pages などのプラットフォームにデプロイする場合は `offline` ブランチを選択するだけで、その他の手順はオンライン版と同じです。詳細は[プロジェクトデプロイ](../deploy)をご覧ください。

## データのインポート・エクスポート

### エクスポート

「設定 > データエクスポート」から、個人のコレクションとカスタムプロンプトを JSON ファイルとしてエクスポートします。

### インポート

以下の形式の JSON ファイルのインポートに対応しています：

- **オフライン版からエクスポートしたファイル**：コレクション、プロンプト、並び順、タグを完全に復元
- **オンライン版からエクスポートしたファイル**：自動的に互換処理
  - ユーザープロンプト → ローカルにマージ（タイトルで重複排除）
  - 厳選コレクション（card）→ ローカルコレクションにマージ
  - コミュニティコレクション（community）→ 自動的にローカルカスタムプロンプトに変換
  - MySpace の並び順 → ローカルに復元
  - カスタムタグ → 追加マージ（既存のものは上書きしない）

### オンライン版からの移行

1. オンライン版（aishort.top）の「マイアカウント」ページからデータをエクスポート
2. オフライン版の「設定」ページから、その JSON ファイルをインポート
3. コミュニティコレクションは自動的にローカルプロンプトに変換され、厳選コレクションは通常通り同期

## よくある質問

### デプロイ後、チームはどう使うのですか？

管理者がイントラネットサーバーにデプロイした後、アクセスアドレス（例：`http://192.168.1.100:3000`）をチームメンバーに伝えるだけです。各個人がブラウザで開けば、インストール不要、登録不要で使用できます。

### 各個人のデータは互いに影響しますか？

影響しません。各個人のコレクションとカスタムプロンプトは各自のブラウザの localStorage に保存され、完全に独立しています。サーバー上には共有のプロンプトライブラリ（読み取り専用）のみがあります。

### データは失われますか？

以下の操作により個人データが失われる可能性があります：

- ブラウザのデータ/キャッシュの消去
- プライベート/シークレットモードでの閲覧
- パソコンやブラウザの変更

重要なデータは定期的に「設定 > データエクスポート」で JSON ファイルとしてバックアップすることをお勧めします。

### チーム間でカスタムプロンプトを共有できますか？

できます。一人が JSON ファイルをエクスポートし、他のメンバーが「設定 > データインポート」からインポートするだけです。インポート時に自動的に重複排除されます。

### プロンプトライブラリを更新するには？

プロンプトライブラリはビルド時にパッケージされた静的データです。更新方法：

1. 管理者が最新の `offline` ブランチコードをプルする
2. 再ビルドしてデプロイ（または最新の Docker イメージをプルする）
3. チームメンバーがブラウザのページを更新すれば新しいコンテンツが表示されます（個人データは影響を受けません）

### オフライン版のデータ形式はオンライン版と互換性がありますか？

互換性があります。エクスポートされた JSON 形式は同じで、2 つのバージョン間で相互にインポートできます。プロンプト ID は 2 つのバージョンで異なります（オンライン版はサーバー ID、オフライン版はタイムスタンプ ID を使用）が、インポート時にタイトルで重複排除されるため、競合しません。
