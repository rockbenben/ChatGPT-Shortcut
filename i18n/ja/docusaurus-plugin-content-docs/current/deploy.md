# プロジェクトのデプロイ

> **対象読者**：AiShort をセルフホストまたはカスタマイズしたい開発者向けです。一般ユーザーは [aishort.top](https://www.aishort.top) をそのままご利用ください。このページを読む必要はありません。

## デプロイメントモデルの選択

ニーズに合ったモデルを選択してください：

| モデル                         | バックエンド                               | 備考                                                                                                                 |
| ------------------------------ | ------------------------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| **スタンダード**（デフォルト） | 公式共有バックエンドを再利用               | フォーク後にサイト名・説明・プロンプトなどをカスタマイズ可能（[設定](./deploy/configuration)参照）。ログイン・お気に入り・コミュニティ・同期はそのまま動作 |
| **オフライン版**               | バックエンドなし、データはブラウザにローカル保存 | ネットワーク隔離された企業・官公庁環境向け。アカウント不要                                                           |
| **完全セルフホスト型バックエンド** | 独自の独立バックエンド                 | 独立したアカウントシステム・完全なデータ所有権・プライベートコミュニティが必要な場合                                 |

最初の 2 つはこのガイドで説明します。3 つ目についてはバックエンドサービスがオープンソースではないため、ユースケースと規模の概要を添えて[開発者にメール](mailto:qingwhat@gmail.com)してください。デプロイプランとサポートを案内します。

## デプロイドキュメント

デプロイの流れに沿って以下に分割しています。必要に応じてご参照ください：

- **[スタンダードデプロイ](./deploy/standard)** —— 公式共有バックエンドを再利用し、ローカルビルド・Vercel・Cloudflare Pages・Docker の 4 つの方法に対応。
- **[オフライン版（社内イントラネット）](./deploy/offline)** —— 企業の社内ネットワークや官公庁ネットワークなど、外部ネットワークにアクセスできない環境向けのオフラインソリューション。バックエンドとアカウント不要。
- **[設定とカスタマイズ](./deploy/configuration)** —— サイトタイトル・説明・プロンプトの変更、およびカスタムバックエンドの接続。
- **[同期更新をオンにする](./deploy/sync-updates)** —— フォークをアップストリームに自動追従させ、機能の遅れを防ぎます。
